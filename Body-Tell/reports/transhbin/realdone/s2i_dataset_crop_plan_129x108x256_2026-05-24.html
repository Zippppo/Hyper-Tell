Body-Tell S2I Dataset Crop Plan

日期 2026-05-24 · 目标尺寸 129x108x256 · 源目录 Body-Tell/S2I-Dataset-70cls/data

目标

新增一个独立裁剪脚本，把 S2I 数据集复制/裁剪到新 dataset package，例如 Body-Tell/S2I-Dataset-70cls-crop-129x108x256/。 不原地覆盖旧 S2I-Dataset-70cls/data。

裁剪对象包括同一体素网格中的 voxel_labels 和由 sensor_pc 对应的空间元数据。训练时标签和体表 occupancy 共用这个 grid。

执行步骤

步骤做什么验证
1 写脚本 Body-Tell/scripts/crop_s2i_dataset.py，参数包含 --input-root, --output-root, --target-shape 129 108 256, --dry-run, --workers。 python -m py_compile Body-Tell/scripts/crop_s2i_dataset.py 通过。
2 先做 dry-run：扫描所有 .npz，计算哪些样本会被 crop、每轴裁剪量、预计输出数量和总大小。 dry-run 输出应接近 shape 报告：train 上约 99.12% 完整覆盖；全量应约 99.13% 完整覆盖。
3 正式生成新目录：复制 dataset_info.json, dataset_split.json, class_presence.json，裁剪并写入新的 data/*.npz。 find OUTPUT/data -name '*.npz' | wc -l 等于输入文件数；旧目录不变。
4 每个样本中心 crop/pad voxel_labels 到 129x108x256，并同步更新 grid_occ_size, grid_world_min, grid_world_max。 随机抽样检查：voxel_labels.shape == grid_occ_size == (129,108,256)，grid_voxel_size 不变。
5 处理 sensor_pc：只保留落在新 grid world bounds 内的点，保证训练 voxelize 后不会依赖已裁掉空间。 抽样检查所有保留点满足 grid_world_min <= sensor_pc < grid_world_max。
6 为新数据集重新生成 class_presence.json，不要直接复用旧 presence，因为 crop 可能改变少数类别 voxel count。 scan_class_presence.py 跑完；num_cases 与 split 总数一致；warnings 可解释。
7 新增配置 phase1_voxtell_body_crop129.yaml，指向新 output root/data/presence，并把 volume_size 设为 [129,108,256]。 train.build_dataset() 能构建 train/val；第一条样本 occupancy 和 target masks 均为目标尺寸。
8 跑 Phase 0/数据一致性验证和测试。 validate_phase0.py 对新数据集通过；pytest Body-Tell/tests -q 通过。
9 跑训练 smoke：小 batch、小 epoch，确认 DataLoader、forward、loss、checkpoint 正常。 python Body-Tell/train.py --config ... --smoke --epochs 1 --batch-size 1 --num-workers 0 退出码为 0。
10 提交 Slurm 短跑：对比当前 164x129x256 和新 129x108x256 的 s/it 与显存。 记录 epoch 前 100 iter 平均 s/it、峰值显存、是否有 OOM/Traceback。

必须避免

不要 原地覆盖 Body-Tell/S2I-Dataset-70cls/data。

不要 复用旧 class_presence.json 作为新训练 presence。

不要 修改 label_vocab.json，否则会影响现有 prompt embedding cache hash。

通过标准

OK 新 dataset package 完整生成，所有 .npz 的 grid shape 都是 129x108x256。

OK 新 presence 与裁剪后标签重新匹配。

OK smoke train 通过，短 Slurm 显示 step time 和显存低于当前配置。
