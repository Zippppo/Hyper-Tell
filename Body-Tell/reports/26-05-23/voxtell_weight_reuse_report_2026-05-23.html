Body-Tell 复用 VoxTell 权重审计报告

日期 2026-05-23 · 范围: Body-Tell/ 当前 Phase 1 模型、VoxTell/ 原始代码、VoxTell v1.1 官方 checkpoint、当前 Body-Tell 训练日志

结论摘要

VoxTell v1.1 的 prompt/text decoder 权重可以复用到 Body-Tell 当前模型。 通过模块名映射后, project_visual_embed、project_text_embed、mask_projections 和 transformer_decoder 都能 100% 按 shape 对齐加载。

按当前 Body-Tell 架构统计, 可从 VoxTell 权重初始化 312,379,424 / 328,980,832 个参数, 约占 95.0%。不能直接加载的是 Body-Tell 自己的轻量 encoder/decoder, 约 16.6M 参数。

推荐加载策略

建议新增一个显式 warm-start 入口, 例如 --init-from-voxtell VoxTell/models-weight/voxtell/voxtell_v1.1, 只加载 prompt decoder 相关权重, 不加载 image encoder/decoder。

prefix_maps = {
    "project_bottleneck_embed.": "project_visual_embed.",
    "project_text_embed.": "project_text_embed.",
    "project_to_decoder_channels.": "mask_projections.",
    "transformer_decoder.": "transformer_decoder.",
}

加载时应该记录:

  • 成功加载的 tensor 数和参数数。
  • 跳过的 VoxTell key 列表或 prefix 统计。
  • Body-Tell 中仍保持随机初始化的 key 列表或 prefix 统计。
  • 若出现同名 shape mismatch, 应直接报错, 不要静默跳过。

主要风险

风险证据影响建议
Transformer forward 语义不完全一致 VoxTell 原始 forward_pre 只做 cross-attention + FFN; Body-Tell 当前 forward_pre 加了 self-attention。 虽然参数 shape 可以加载, 但权重语义不完全等价。直接加载全部 transformer 权重可能引入不稳定 warm-start。 若目标是最大化复用 VoxTell 权重, 建议把 pretrained 实验分支的 transformer 语义对齐 VoxTell 原始实现。
Transformer activation 不一致 VoxTell 构造 TransformerDecoderLayer(..., normalize_before=True) 时使用默认 activation relu; Body-Tell 当前显式使用 gelu。 FFN 权重 shape 相同, 但非线性不同会改变已训练表示。 pretrained A/B 实验中建议使用 relu 以贴近 VoxTell 权重训练环境。
Visual domain 不同 VoxTell 输入为医学影像 intensity patch; Body-Tell 输入为 occupancy grid。 image encoder/decoder 权重跨域复用收益不确定, 且当前结构不对齐。 第一阶段只复用 prompt decoder; 不加载 VoxTell encoder/decoder。
Body-Tell 实际 fusion stage 数和配置名有漂移 配置写 num_maskformer_stages: 5, 但当前 5 个 encoder channels 只能产生 4 个 upsampling/fusion stages。 容易误判复用范围。VoxTell 的 5 个 projection 中只有前 4 个映射到 Body-Tell 当前模型。 后续可把字段改名为 num_fusion_stages, 或在超出上限时显式 warning。

除权重外值得复用的 VoxTell 内容

内容是否已经复用建议
Qwen prompt embedding 方式 基本已复用 Body-Tell 使用相同 text encoder、instruction、last-token pooling 和 2560 维 embedding。 保留当前实现。后续只需要确保 prompt cache metadata 严格记录 model name、instruction、vocab hash。
Prompt decoder + multi-stage mask projection 应优先复用 当前 shape 已对齐。 作为 warm-start 初始化方案落地, 并做 A/B 实验。
Sliding-window inference、pad/crop、Gaussian aggregation 尚未复用 有价值, 但优先级低于权重 warm-start。Body-Tell 输入是 occupancy grid, 不能直接照搬医学影像 normalization。
原始 nnU-Net ResidualEncoder/Decoder 不建议直接复用 当前 Body-Tell 已经能训练, 且数据域不同。除非计划做更大的架构回归实验, 否则不要作为第一优先级。

推荐实验计划

  1. 新增 --init-from-voxtell 参数和一个独立的 partial loader, 只加载 prompt decoder 映射权重。
  2. 新增一个 pretrained 配置, 明确记录 transformer_activation: relu 和 voxtell_forward_compat: true。
  3. 保持当前 slurm/body-tell-phase1.sh 的 batch size、lr、loss、data split 不变, 只比较初始化差异。
  4. 跑两组 A/B:
    • Baseline: 当前随机初始化 Body-Tell, 当前 best val foreground Dice 0.3257。
    • Warm-start: 加载 VoxTell prompt decoder 权重, encoder/decoder 随机初始化。
  5. 对比 early convergence、best val foreground Dice、negative fp rate 和 per-class Dice。若 warm-start early convergence 明显更快但最终 Dice 无提升, 再考虑冻结/分层学习率。

最终判断

可以复用, 但要按 prompt decoder warm-start 复用, 不要整网强行加载。

从参数覆盖率看, VoxTell v1.1 对 Body-Tell 当前模型非常有价值: 约 95% 参数可以初始化, 且这些参数正是文本条件分割任务最关键的 prompt decoder 部分。从工程风险看, 第一版实现需要同时解决 transformer forward 语义差异, 否则 shape 匹配并不等于行为等价。
