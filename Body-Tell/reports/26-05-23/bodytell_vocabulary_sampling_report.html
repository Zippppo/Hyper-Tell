Body-Tell Vocabulary 与 Prompt Sampling 代码审计

范围：Body-Tell/ 当前 Phase 1 训练链路、VoxTell/ 本地实现与文档中记录的训练策略。生成日期：2026-05-23。

结论先行

当前 Body-Tell 的训练确实有 prompt 采样，但粒度是“先按 case 抽类别，再对每个被抽中的类别抽 1 条 prompt”。默认配置每个 case 目标是 2 positive + 1 negative，不是遍历所有类别，也不是同一类别一次采多条 prompt。
  • 每个被选中的类别只采 1 条 prompt：实现位于 body_tell/data/vocabulary.py:328-338，对每个 positive/negative class 使用 rng.choice(grouped[class_id])。
  • 采样对同一个 case 是固定的：HyperBodyPromptDataset.__getitem__ 在 body_tell/data/dataset.py:110 使用 random.Random(self.seed + index)。因此同一个 index 每个 epoch 都得到同一组类别和同一条 prompt。
  • VoxTell 公开仓库里没有训练期 dataloader/sampler 源码，本地可见代码主要是推理和模型 forward；本报告对 VoxTell 训练采样的描述来自本地文档对论文 Appendix A 的摘录。
  • Body-Tell 当前比 VoxTell 原训练策略少了几个关键动态性：epoch 级重新采样、85% foreground patch oversampling、显式 25/75 主名/同义词采样、验证时全 label 遍历。

Body-Tell Vocabulary 是怎么实现的

Vocabulary 是一个静态 artifact，而不是训练时动态构造。核心路径如下：

文件职责关键点
Body-Tell/configs/label_vocab.json最终词表每个 class 有 id、source_name、canonical、prompts、train_as_positive、eval_as_foreground；另有 10 个 aggregate concept。
Body-Tell/scripts/build_label_vocab.py从 dataset_info.json 生成词表普通 class 多为 3 条 prompt，rib class 是 4 条 prompt；class 0 是 inside_body_empty，被排除在 foreground 训练外。
Body-Tell/scripts/build_prompt_embedding_cache.py离线生成 prompt embedding用同一条 instruction 包裹 prompt，再用 Qwen3-Embedding-4B 做 last-token pooling；输出 prompt_embeddings.pt。
Body-Tell/body_tell/data/vocabulary.py词表读取、校验、flatten、采样flatten_prompt_records() 给每条 prompt 分配稳定 index；embedding cache 通过 vocab hash 和 prompt records 严格校验。

生成 embedding 时的模板与 VoxTell 推理代码一致：

Instruct: Given an anatomical term query, retrieve the precise anatomical entity and location it represents
Query: {prompt}

flatten_prompt_records() 会同时 flatten class prompts 和 aggregate prompts，所以 embedding cache 中有 264 条记录。但训练 sampler 使用 prompt_records_by_class(... include_aggregates=False)，因此 aggregate prompt 当前只是被缓存，不会参与 Phase 1 训练。

counts = case_record["voxel_counts"]
trainable = {class_id | train_as_positive}
present = {class_id | class_id in trainable and voxel_count >= min_voxels}
absent = trainable - present

positive_classes = sample_without_replacement(present, num_positive)
negative_classes = sample_without_replacement(absent, num_negative)

for class_id in positive_classes:
    record = rng.choice(prompt_records_for_this_class)  # 只取 1 条
    target_empty = False

for class_id in negative_classes:
    record = rng.choice(prompt_records_for_this_class)  # 只取 1 条
    target_empty = True
回答你的判断：“当前采样 prompt 好像只是一个类别采样一个？”基本正确。更精确地说：每个 case 先抽最多 2 个 positive 类别和 1 个 negative 类别；每个被抽中的类别只选 1 条 prompt。由于类别采样是 without replacement，同一个 case 内不会对同一类别同时训练多个同义 prompt。

现象影响证据

注意negative 有时缺失 如果一个 case 的 presence 已覆盖全部 69 个 trainable classes，则 absent 为空，最终只返回 2 条 positive prompt。当前 train split 有 290 个这种 case。 _sample_without_replacement() 对空列表返回 []；复算统计：train sample count 分布 {2: 290, 3: 9489}。

VoxTell 本地实现怎么处理 prompt

本地 VoxTell/ 仓库可见代码主要是 inference 与 model，不包含训练 dataloader/sampler 源码。因此要分成两层看：

1. 公开代码中可直接确认的行为

文件行为
VoxTell/voxtell/inference/predictor.py:173-205embed_text_prompts() 接受单条或多条 free-text prompt，在线用 Qwen3-Embedding-4B 转成 (1, num_prompts, 2560) embedding。
VoxTell/voxtell/utils/text_embedding.py:14-20每条 prompt 使用和 Body-Tell 相同的 instruction 包裹。
VoxTell/voxtell/model/voxtell_model.py:258-267模型 forward 对 num_prompts 循环，每条 prompt 生成自己的 mask embedding，再 concat 成 (B, N, ...) 输出。
VoxTell/readme.md:127-199CLI/API 支持用户传入任意 prompt list；默认每条 prompt 输出一个二值 mask，也可以合并成多标签文件。

也就是说，VoxTell 推理阶段没有固定 vocabulary，也没有 sampler；prompt list 由用户直接给。

迁移完整度判断

建议

  1. 如果目标是更接近 VoxTell，先修正 epoch-level sampling。 给 dataset 增加 set_epoch(epoch) 或在 sampler 中引入 epoch/global step，使同一个 case 在不同 epoch 能看到不同 positive class、negative class 和 synonym。
  2. 把同义词采样概率显式化。 例如 canonical_prob=0.25，先决定是否用 canonical，再从非 canonical prompts 中随机抽。这样和 VoxTell 文档策略一致，也便于 ablation。
  3. 增加 exhaustive validation。 保留 sampled val 作为快速指标，同时新增一个验证模式：对每个 case 遍历全部 present trainable classes 的 canonical prompt，输出 class-wise Dice/coverage。
  4. 明确 aggregate 是否要训练。 如果需要训练 kidneys、ribs 这类组合概念，当前 build_target_mask() 已支持多个 class id，主要缺 sampler 分支和采样比例。
  5. 记录 negative 缺失 case。 当前 290 个 train case 没有 absent class 可采 negative。可以接受，但建议在日志中统计，避免以为所有 batch 都严格 2+1。

一句话总结

Body-Tell 已经把 VoxTell 的“text prompt 条件分割”主链路迁过来了，但当前 prompt sampler 更像一个 deterministic per-case sampler：每个被抽中的类别只用一条 prompt，且同一 case 跨 epoch 不变。若要复刻 VoxTell 训练策略，最优先要补的是 epoch 级随机重采样、显式 synonym 采样比例和更完整的验证覆盖。
