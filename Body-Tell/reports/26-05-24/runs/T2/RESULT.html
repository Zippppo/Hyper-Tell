T2 Result - Fix prompt metrics foreground Dice aggregation

Task Summary

Implemented nan-safe epoch foreground Dice aggregation for Body-Tell prompt metrics. Foreground Dice now exposes valid positive prompt sum/count from compute_prompt_metrics(), and train_one_epoch() / evaluate() aggregate by those counts. Batches without valid positive prompts contribute zero foreground Dice count, so they do not poison the epoch value with nan.

Tests Changed

  • Added Body-Tell/tests/test_prompt_loss_backward.py::test_prompt_metrics_nan_safe_foreground_dice_aggregation.
  • The test runs a two-batch training loop: one positive prompt batch with near-perfect Dice, followed by a negative-only batch with a false positive. It asserts the epoch foreground Dice stays finite and high, while negative false-positive rate remains nonzero.
