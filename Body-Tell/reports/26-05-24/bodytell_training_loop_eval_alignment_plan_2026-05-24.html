本文件的角色

SP-C 只负责训练机制与评估协议，不负责模型结构迁移，也不负责数据采样器和 vocabulary 扩充。结构问题继续由 SP-A 负责，数据构造问题继续由 SP-B 负责。

SP-C 的任务是把父报告中已经确认的训练侧差异变成可落地步骤：固定 update-count 训练节奏、SGD + polynomial decay、5-scale deep supervision、可精确恢复的 checkpoint、prompt-aware 评估协议，以及能暴露这些行为的测试矩阵。

本计划不把当前默认训练全部强制切到 paper-aligned profile。先新增显式 profile 和代码路径，保留当前 AdamW/短训配置作为回退实验；只有在 smoke 和 resume 验收通过后，才应把长训入口切到 paper-aligned profile。

前置依赖

依赖 原因 SP-C 处理
SP-A P3：5-stage VoxTellDecoder 5-scale deep supervision 需要模型稳定返回 5 个尺度输出。 Task 5 在 SP-A P3 之前只能写配置和 loss 测试骨架，不启用长训。
SP-B Step 1：动态采样 RNG 训练 loop 每个 epoch 必须同时驱动 sampler 和 dataset epoch state。 Task 1 要调用 dataset epoch setter；若 dataset 尚未实现，则记录 warning 并在测试中暴露。

实施任务

Task 0. 新增 paper-aligned profile 骨架

  1. 新增 Body-Tell/configs/phase1_voxtell_paper_aligned.yaml，显式声明 profile: paper_aligned 或等价字段。
  2. 集中写入训练语义：iterations_per_epoch: 250（机制对齐，固定每 epoch 的 update 数）、optimizer: sgd、lr: 1e-4、lr_schedule: poly、deep_supervision: true、deep_supervision_weights: [1, 0.5, 0.25, 0.125, 0.0625]。
  3. 训练预算字段 epochs 与 total_updates 不写死论文 2000 / 500000。这两个字段由实际算力倒推决定：先估单 step 墙钟时间 t_step 和可承受训练时长 T_budget，再令 total_updates = floor(T_budget / t_step)、epochs = ceil(total_updates / iterations_per_epoch)。profile 中必须以注释或独立字段（如 budget_source: "wall_clock=Xh, t_step=Ys"）显式注明此预算来源，避免被误读为复刻论文。详见父文档 总控计划 中 2026-05-24 决议。
  4. 模型字段应引用 SP-A aligned 架构：architecture: voxtell_v1_1、num_maskformer_stages: 5。在 SP-A 未完成前，该 profile 可以存在但不作为默认训练入口。
  5. 训练启动日志和 W&B config 中打印 paper_aligned profile、总 update 数、optimizer/scheduler、deep supervision 权重。

验收：读取配置时能区分当前 baseline profile 与 paper-aligned profile；未启用 SP-A aligned 架构时，长训入口不能静默落回旧轻量模型。

Task 1. 固定 update-count 训练 loop

  1. 新增 iterations_per_epoch 支持；当该字段存在时，一个 epoch 固定执行指定数量的 optimizer updates，不再等价于完整遍历 DataLoader。
  2. 实现可复用的无限 dataloader iterator；迭代耗尽后重建 iterator，但 epoch 和 global_step 语义保持连续。
  3. 日志同时记录 epoch、iteration_in_epoch、global_step、total_updates、samples_seen。
  4. 每个 epoch 调用 DistributedSampler.set_epoch(epoch)；如果 dataset 提供 set_epoch(epoch)，同步调用它，保证 SP-B 动态 prompt RNG 能随 epoch 更新。
  5. smoke 模式允许覆盖为小数值，例如 epochs=2、iterations_per_epoch=2，但仍走同一 fixed-update 代码路径。

验收：2-epoch smoke 的 optimizer step 数严格等于 epochs * iterations_per_epoch；日志中的 global_step 单调递增且无跳号。

Task 2. 配置驱动 optimizer profile

  1. 把 optimizer 构造从硬编码 torch.optim.AdamW(... weight_decay=1e-5) 改为配置驱动 factory。
  2. paper-aligned profile 使用 torch.optim.SGD，初始 LR 为 1e-4。SGD 的 momentum、weight_decay 必须来自配置并写入日志；如果无论文/官方依据，不应把 AdamW 的 weight decay 直接迁移过来。
  3. baseline profile 保留 AdamW，避免无法回退或无法做 ablation。
  4. checkpoint 中写入 optimizer 名称、param group 数、各 group 的初始 LR 与当前 LR。

验收：同一训练脚本能通过配置切换 AdamW 与 SGD；paper-aligned profile 启动日志中明确显示 optimizer=sgd、lr=1e-4。

Task 3. Polynomial LR schedule

  1. 新增 poly scheduler，按 optimizer update 数计算进度：progress = global_step / total_updates。
  2. LR 在每次 optimizer step 后更新，而不是每个 epoch 后更新。
  3. poly_power 进入配置并写入日志；若采用 nnU-Net 惯例值，应在配置中显式声明，避免隐藏默认。
  4. scheduler state 必须能从 checkpoint 恢复；恢复后同一 global_step 的 LR 应与不中断训练完全一致。

验收：scheduler 数值单测覆盖 step 0、半程、末程和 resume 后一步；LR 不应在 epoch 边界出现额外跳变。

Task 4. Checkpoint 与精确恢复

  1. checkpoint 写入 model_state_dict、optimizer_state_dict、scheduler_state_dict、epoch、iteration_in_epoch、global_step、total_updates、AMP scaler、配置 hash、architecture 字段。
  2. 恢复入口必须在 dataloader / sampler / dataset epoch state / scheduler state 全部恢复后继续训练。
  3. 保存 best checkpoint 和 latest checkpoint；latest 用于 resume，best 用于评估与发布。
  4. 恢复日志打印 checkpoint 中的 global_step、下一个 epoch / iteration_in_epoch、当前 LR、配置 hash 是否匹配。

验收：构造 interrupted run 与 uninterrupted run，在同一 global_step 比较 LR 和 scheduler state，必须完全一致。

Task 5. 启用 5-scale deep supervision loss

  1. 依赖 SP-A P3：模型必须返回 5 个尺度输出，且最高分辨率输出仍与 target mask 的轴语义一致。
  2. paper-aligned profile 设置 deep_supervision: true 与权重 [1, 1/2, 1/4, 1/8, 1/16]。
  3. loss 层保存原始权重，并在日志中同时报告归一化后的有效权重；避免“论文权重”和“实现归一化”混淆。
  4. 每个 scale 的 target 使用统一 resize 策略；日志至少记录总 loss、BCE、Dice，以及可选的 per-scale loss。
  5. 补充反传测试：5 个尺度输出都参与 loss，且每个可训练输出分支都有梯度。

验收：1-batch backward 通过；deep supervision 打开时返回 5-scale loss，关闭时仍能跑 baseline；SP-A P3 未完成时不得把 paper-aligned 长训标记为通过。

Task 6. 评估协议扩展

  1. 保留现有 S2I val foreground mean Dice 与 negative FP，作为本地域内基线。
  2. 新增 prompt variants 评估：同一 class 用 canonical 与 variant prompt 分组统计 Dice。
  3. 新增 negative empty-mask 评估：对 patch/case absent prompt 统计 FP voxel、FP rate、empty-mask success rate。
  4. 新增 held-out class/concept 分组评估；如果当前 split 没有正式 held-out concept，先用配置化列表声明，不在报告中伪造论文级 zero-shot 结论。
  5. 临床 report-derived prompts 与 ReXGroundingCT fine-tuning 只登记为论文协议缺口；在 Body-Tell 缺少对应数据前，不作为本轮强制验收。

验收：评估报告能按 known concepts、prompt variants、negative prompts、held-out classes/concepts 分组输出 Dice/FP，并记录使用的 prompt records / vocab hash。

Task 7. Paper-alignment 测试矩阵

测试 覆盖风险 验收口径
CPU shape test fixed-update loop、5-scale 输出、非立方输入轴语义。 forward 输出 shape 与 target mask 一致；deep supervision 返回 5 个尺度。
1-batch backward SGD、deep supervision、AMP off 的基础反传。 loss finite；optimizer step 后至少一个参数变化。
2-epoch smoke iterations_per_epoch、日志、checkpoint 写入。 step 数等于 epochs * iterations_per_epoch，latest checkpoint 可读取。
scheduler 数值测试 poly decay 按 update 而非 epoch 更新。 指定 step 的 LR 与公式一致。
resume 等价测试 checkpoint/scheduler/iteration 恢复。 interrupted run 与 uninterrupted run 在同一 step 的 LR 完全一致。
prompt 采样分布测试 SP-B 动态 RNG 与 25/75 采样是否被 train loop 正确驱动。 同一 case 跨 epoch prompt 可变化；positive canonical 比例接近配置。
deep supervision 反传测试 5 个尺度是否都参与优化。 每个输出分支产生非空梯度或等价可观测信号。

落地顺序

  1. 先做 Task 0，让 paper-aligned profile 有显式配置入口，但不改变默认训练。
  2. 随后做 Task 1、Task 2、Task 3，形成 fixed-update + SGD + poly 的最小训练闭环。
  3. 再做 Task 4，确保中断恢复不会破坏 poly schedule。
  4. SP-A P3 完成后做 Task 5；SP-B Step 1/2 完成后补齐 Task 6 的最终验收口径。
  5. Task 7 贯穿每个阶段：每完成一项机制，就补对应测试，避免把长训当成唯一验收。

SP-C 的关键路径是 Task 0 → Task 1/2/3 → Task 4 → Task 5。Task 6 可先做报告框架，但最终指标必须等 SP-B patch-level sampler 和 vocabulary/cache 对齐完成后再定稿。

最终判定

SP-C 已接管父报告中所有训练机制与评估协议缺口。它不替代 SP-A/SP-B，而是在两者提供 aligned model 和 patch-level sampling 后，把训练节奏、optimizer/scheduler、deep supervision、checkpoint resume 与评估协议接成完整 paper-aligned 训练闭环。

本文件仅创建子计划，未修改训练代码。
