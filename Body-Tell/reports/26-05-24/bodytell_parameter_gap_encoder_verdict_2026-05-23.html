Body-Tell 参数差距归因结论报告

生成日期：2026-05-23；依据： body_tell_parameter_audit.html、 bodytell_encoder_backbone_report_2026-05-23.html、 以及 Body-Tell / VoxTell 源码。

结论

主要参数量差距确实在 encoder 上：Body-Tell 与 VoxTell 总参数差距约 111.05M，其中 encoder 差距约 81.80M，占总差距约 73.67%。

但不能严格表述为“剩下模块完全一致”。更准确的说法是：visual projection、text projection、transformer decoder 的参数量一致；mask projection 和 decoder 因 stage 数不同仍有明显差距；transformer 代码路径也存在行为差异，尽管参数量相同。

参数差距分解

模块 Body-Tell VoxTell 差值 占总差距 判断
encoder 8.49M 90.30M +81.80M 73.67% 最大差异来源。Body-Tell 是 5-stage 轻量普通 3D CNN；VoxTell 是 6-stage residual encoder。
mask projection 46.23M 71.40M +25.18M 22.67% 不是完全一致。Body-Tell 实际只有 4 个 projection；VoxTell 有 5 个 projection。
decoder own params 8.11M 12.18M +4.07M 3.66% 不是完全一致。VoxTell 多一个 stage，因此 decoder 参数更多。
visual / bottleneck projection 4.85M 4.85M 0 0% 参数量一致。两边选中 visual channel 都是 320，再投影到 2048。
text projection 9.44M 9.44M 0 0% 参数量一致。两边都是 2560 -> 2048 -> 2048。
transformer decoder 251.86M 251.86M 0 0% 参数量一致。两边同为 6 层、2048 hidden、8 heads。

为什么说主要差距在 encoder

Body-Tell 的 encoder 在 Body-Tell/body_tell/models/voxtell_body_model.py 中由 _BodyEncoder 构建：5 个 stage，每个 stage 是一个 _ConvBlock，每个 block 只有两层 Conv3d + GroupNorm + GELU，没有 residual skip。

VoxTell 的 encoder 在 VoxTell/voxtell/model/voxtell_model.py 中使用 dynamic_network_architectures 的 ResidualEncoder。按照 v1.1 plans.json，它有 6 个 stage，通道为 [32, 64, 128, 256, 320, 320]，残差块数为 [1, 3, 4, 6, 6, 6]。

因此二者不是同一个 backbone 的小幅配置差异，而是浅层普通 CNN 与 nnU-Net-style residual feature pyramid 的结构差异。仅 encoder 一项，VoxTell 就比 Body-Tell 多约 81.80M 参数。

哪些模块可以认为基本一致

模块 参数量是否一致 实现/行为是否完全一致 说明
visual projection 是 基本等价 Body-Tell 的 project_visual_embed 对应 VoxTell 的 project_bottleneck_embed，参数结构相同。
text projection 是 基本等价 两边都是两层 Linear，中间 GELU，维度为 2560 -> 2048 -> 2048。
transformer decoder 是 不完全一致 参数规模相同；但 Body-Tell 的 pre-norm 路径包含 self-attention，而 VoxTell 当前 forward_pre 路径只执行 cross-attention + FFN。
mask projection 否 不完全一致 Body-Tell 因 encoder 只有 5 个 channel stage，实际 fused stage 数为 4；VoxTell 为 5。
decoder 否 不完全一致 Body-Tell 是手写 _VoxTellBodyDecoder；VoxTell decoder 依赖 encoder 的 output_channels、strides、kernel_sizes，并随 6-stage encoder 构建 5 个 decoder stage。

推荐表述

可以说：“Body-Tell 和 VoxTell 的总参数差距主要来自 encoder，encoder 差距约 81.80M，占总差距约 73.67%。文本投影、视觉投影和 transformer decoder 的参数量保持一致。不过剩余模块不能说完全一致：Body-Tell 因少一个 encoder/decoder stage，mask projection 少一级，decoder own params 也更少；此外 transformer 实现虽然参数量一致，但 pre-norm 行为并非完全相同。”
