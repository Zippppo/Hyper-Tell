Body-Tell Sampling 与 Vocabulary 优化思路

范围：Body-Tell/ 当前 Phase 1 S2I 配置、既有采样审计报告、既有 vocabulary audit、VoxTell/VoxTell.pdf Supplementary A.3 与 C。生成/修订日期：2026-05-24；本版按重新配置后的 S2I-Dataset-70cls 复算。

结论

本版报告只保留 VoxTell 论文/补充材料明确提到的训练数据构造内容：2 positive + 1 negative prompt、85% foreground patch oversampling、25/75 主名/改写 prompt 采样、训练前预计算 text embeddings、vocabulary expansion / harmonization / expert validation、combined-label alternatives。此前报告里属于额外工程增强的 class-balanced positive、hard-negative、contrastive co-sampling、exhaustive validation 不再作为实现目标。
  1. 第一优先级是 sampler 对齐：当前 Body-Tell 虽然配置了 2+1 prompt，但采样单位是整 case，且同一个 case 跨 epoch prompt 固定；VoxTell 的描述是训练 patch 上动态采 2 positive + 1 negative。
  2. 第二优先级是 patch 对齐：VoxTell 明确使用 85% foreground patch oversampling；Body-Tell 当前没有 patch sampler，因此 present/absent 判断不在局部 patch 上发生。
  3. 第三优先级是 prompt/vocabulary 对齐：VoxTell 明确 25% 默认主名、75% 改写变体，并通过 vocabulary construction 生成至少 5 个 validated variants 和 combined-label alternatives。
  4. 当前数据口径：本报告以当前 S2I-Dataset-70cls/class_presence.json 为准，共 11,070 cases，train/val/test 为 10,070/500/500。

实现顺序

顺序很重要：先修训练采样，再扩 vocabulary。否则新增 prompt 仍会被当前 deterministic per-case sampler 固定下来，无法体现 VoxTell 的随机 prompt 训练策略。

Step 1. 让训练时 prompt 采样真正随机

对应 VoxTell：Supplementary A.3 使用 sampled positive/negative prompts，并随机采 synonyms/rephrasings。Body-Tell 当前同一 index 跨 epoch 固定，因此第一步是把随机性从 seed + index 改成训练轮次相关。

  1. 给 HyperBodyPromptDataset 增加 set_epoch(epoch) 或等价机制。
  2. 采样种子使用 base_seed + epoch * large_prime + index；DDP 时确保各 rank 复现但不冲突。
  3. 训练循环每个 epoch 调用 dataset 的 epoch setter。
  4. 验收：同一个 case 在两个 epoch 中采到的 prompt 文本可以变化；不再是固定 prompt-conditioned sample。

Step 2. 把训练单位从整 case 改成 patch

对应 VoxTell：输入训练 patch，并明确 foreground structures 以 85% 概率过采样。Body-Tell 当前在整 case 上统计 present/absent，不符合 VoxTell 的 patch 语境。

  1. 新增训练配置：patch_size、foreground_oversample_prob: 0.85。
  2. 在 __getitem__ 中先读取整 case，再裁出训练 patch，后续 target mask 和 present/absent 都基于这个 patch。
  3. 85% 概率裁到包含 foreground voxel 的 patch；15% 概率随机裁 patch。
  4. 验收：训练日志中 foreground crop 比例接近 85%，random crop 比例接近 15%。

Step 3. 在 patch 内采 2 positive + 1 negative

对应 VoxTell：每个训练 patch 查询 3 条 prompt：2 条来自当前 patch 中 present structures，1 条来自 absent structure。Body-Tell 当前是在整 case presence 上采样。

  1. 对裁出的 patch 重新统计 foreground class voxel counts。
  2. positive classes 从 patch-present classes 中采 2 个。
  3. negative class 从 patch-absent trainable classes 中采 1 个。
  4. negative target 仍为空 mask；positive target 是对应 class mask。
  5. 验收：完整 3 prompt 样本比例应显著高于当前 train split 的 9,520/10,070；negative 缺失应明显下降。

Step 4. 对 positive prompt 实现 25/75 主名/改写采样

对应 VoxTell：positive text prompts 中，默认主名 25%，同义词/改写 75%。这一步只声明 VoxTell 明确写到的 positive prompt 规则，不引入 hard-negative 或额外对比采样。

  1. 每个 prompt record 保留 is_canonical。
  2. 对 positive class：先以 25% 概率选 canonical prompt，否则从非 canonical variants 中随机选。
  3. 如果某个 class 暂时没有非 canonical variant，则回退到 canonical，并在审计中标记。
  4. 验收：positive prompt 的 canonical 比例接近 25%，variant 比例接近 75%。

Step 5. 按 VoxTell Appendix C 扩 vocabulary

对应 VoxTell：semantic label expansion、cross-dataset harmonization、human expert validation、每个 concept 至少 5 个 variants、combined-label alternatives。Body-Tell 不需要复制 VoxTell 的 1,087 concepts 规模，但应复制构造原则。

  1. 对 70 个 base classes 做 semantic label expansion，每类至少 5 条 validated prompts。
  2. 保留 laterality 自包含原则：所有 left/right prompt 必须包含结构名，不能只写方向词。
  3. 建立 combined-label alternatives：如 kidneys、ribs、left ribs、right ribs、gluteal muscles 等，只使用现有 labels 可以 union 得到的组合。
  4. 做人工/专家审校，移除与真实 mask 语义不一致的 prompt，例如过度暗示 wall-only、parenchyma-only、subpart-only 的表达。
  5. 验收：每个可训练 base class 至少 5 条 prompt；combined labels 有明确 component class ids；静态 vocabulary validation 无错误。

Step 6. 重新预计算 text embeddings

对应 VoxTell：训练时所有 text embeddings 预计算。Body-Tell 已有 cache 机制，所以 vocabulary 变更后必须重建 cache。

  1. 使用同一 instruction：Given an anatomical term query, retrieve the precise anatomical entity and location it represents。
  2. 用 Qwen3-Embedding-4B 对所有 class prompts 和 combined-label prompts 重新编码。
  3. 写入 prompt_embeddings.pt，并更新 vocab hash / prompt records 校验。
  4. 验收：embedding shape 与 prompt records 数量一致，cache hash 与 label_vocab.json 一致。

Vocabulary 扩增边界

Vocabulary 可以扩，但只能按 VoxTell Appendix C 的原则扩：同义/改写、语义统一、combined labels、人工审校。不能引入 S2I 没有 mask 支撑的新监督类别，也不能加入真实 mask 不支持的局部/病灶描述。
允许扩增原因边界
同一 class 的主名、同义词、改写表达 VoxTell 明确使用 synonyms/rephrasings，并要求至少 5 个 variants。 表达必须仍指向同一个完整 label mask。
combined-label alternatives VoxTell Appendix C 明确生成 clinically meaningful groupings。 只能组合现有 class ids，例如左右器官、rib groups、肌肉组。
拼写/术语变体 VoxTell 评估关注 synonyms、rephrasings 和拼写扰动的鲁棒性。 少量加入，且不能产生歧义缩写。
语义边界审校 VoxTell pipeline 包含 harmonization 和 human expert validation。 删除与数据集真实标注范围冲突的 prompt。

明确不纳入本轮

  • 没有 S2I mask 的新监督 class，例如 tumor、vessels、aorta、left lung/right lung。
  • 没有 sub-mask 支撑的局部描述，例如 upper pole kidney、anterior rib、Couinaud segment。
  • 论文没有明确写的采样增强：class-balanced positive、hard-negative、contrastive co-sampling。

实现注意点

  • 扩词必须配套重建 embedding cache：当前 cache 对 prompt records 做 hash 校验，改 label_vocab.json 后旧 cache 会失效。
  • 采样比例要进 config：foreground_oversample_prob=0.85、canonical_prob=0.25、num_positive=2、num_negative=1 应可复现实验。
  • combined labels 不能污染 base class：组合概念是额外 prompt target，不应把 base label 的 canonical 名称改成群组名。
  • patch present/absent 是核心：negative 应基于 patch absent，而不是 whole-case absent。
  • 严格限定范围：本轮不实现论文未明确写到的 class-balanced、hard-negative、contrastive co-sampling。

最终判断

Body-Tell 靠近 VoxTell 的最短路径不是加入更多自定义技巧，而是严格复刻 VoxTell 明确写到的数据构造：训练时动态采 prompt、patch 内 2 positive + 1 negative、85% foreground patch oversampling、positive prompt 25/75 主名/改写、Appendix C 风格 vocabulary expansion 与 combined-label alternatives、训练前预计算 text embeddings。其他增强项应等这些基础对齐后再单独讨论。
