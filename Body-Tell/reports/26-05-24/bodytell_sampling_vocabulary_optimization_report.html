Body-Tell Sampling 与 Vocabulary 优化思路

范围：Body-Tell/ 当前 Phase 1 S2I 配置、既有采样审计报告、既有 vocabulary audit、VoxTell/VoxTell.pdf Supplementary A.3 与 C。生成日期：2026-05-24。

结论

要缩小 Body-Tell 与 VoxTell 的训练分布差距，优先级应是：先修 sampler，再有边界地扩 vocabulary。当前 70 类 vocabulary 静态校验是干净的，但 prompt 密度、组合概念训练、epoch 级重采样、patch 级 foreground/negative 机制都不足。单纯往 label_vocab.json 里堆词，不会自动改善训练，因为当前 sampler 每个被抽中的类别仍只取 1 条 prompt，并且同一个 case 跨 epoch 不变。
  1. 采样层：补 epoch-level 动态采样、显式 25/75 主名/同义词概率、patch/ROI 级 85% foreground oversampling、class-balanced positive、hard-negative/contrastive co-sampling、exhaustive validation。
  2. 词表层：不建议盲目扩到 VoxTell 的 1,087 concepts；Body-Tell 只有 70 个可监督标签。更合理的是把 70 个 base class 扩到每类至少 5 条经审校 prompt，并把已有 10 个 aggregate concept 纳入训练。
  3. 审计层：已有 vocabulary_audit.html 是有效参考，但它记录的 case 规模是 10,779；当前 active S2I 配置实际使用 8,176 cases。因此后续对覆盖率、small/rare class、negative 缺失比例的审计应重新绑定到 S2I-Dataset-70cls。

采样优化方案

1. 先让采样真正随 epoch 变化

给 dataset 增加 set_epoch(epoch)，训练循环在每个 epoch 前调用；采样随机种子使用 seed + epoch * large_prime + index，DDP/worker 场景再混入 rank 或 worker seed。这样同一 case 在不同 epoch 能看到不同 positive class、negative class 和同义 prompt。

新增配置建议默认目的
sampling.dynamic_epochtrue关闭当前 deterministic per-case 行为。
sampling.canonical_prob0.25复刻 VoxTell 的主名/改写概率。
sampling.log_histogramstrue每 epoch 记录 class、prompt、canonical、negative 缺失比例。

2. 把整 case 训练升级为 patch/ROI 训练

当前 negative 缺失的根因是整 case 常常包含几乎全部 foreground classes。VoxTell 的 negative 是 patch 语境下的 absent structure；一个结构即便在全身 case 里存在，也可以在当前 crop 里不存在。因此更接近 VoxTell 的方案是把 present/absent 的计算放到 crop 后。

  1. 85% 概率先选一个 foreground class，再围绕其 voxel bounding box 或随机 foreground voxel 取 crop。
  2. 15% 概率随机取 crop，保留背景和稀疏结构分布。
  3. 在 crop 内重新统计 present classes，然后采 2 positive + 1 negative。
  4. 如果 crop 内 positive 不足 2，允许退化到 1 positive + 2 negative 或重新采 crop；该策略应显式计数。
如果短期不做 patch sampler，至少应做 class-balanced positive 采样和 negative 缺失日志。但这只能缓解覆盖不均，不能真正复刻 VoxTell 的 85% foreground patch distribution。

3. 让 small/rare class 有更稳定的正样本梯度

当前 S2I presence 显示 40 个 small structures；典型包括 adrenal glands、esophagus、trachea、duodenum、thyroid、ribs、sternum、scapula、clavicle、gluteus minimus 等。建议 positive class 采样不要只在 present classes 中均匀抽，而是加入 class weight：

权重来源建议公式备注
case frequencyw_case = 1 / sqrt(case_count + eps)提升脑、颅骨、锁骨、肱骨等低覆盖类。
median target volumew_size = 1 / sqrt(median_voxels + eps)提升 adrenal、rib、esophagus 等小目标。
manual capw = clamp(w_case * w_size, max=cap)避免极小类过采样导致训练分布过窄。

4. 引入 hard-negative 与 contrastive co-sampling

随机 absent negative 太容易，尤其对 laterality 和 rib ordinal 这类细粒度标签帮助有限。建议增加两类采样：

  • Absent hard negative：当 left clavicle present 且 right clavicle absent 时，优先把 absent side 作为 negative；对 ribs 优先采 adjacent ordinal 或 opposite side。
  • Confusable co-positive：当左右器官或相邻 ribs 同时 present 时，优先把它们放在同一 prompt group 中作为不同 positive masks，让模型在同一图像上下文里学习边界和 laterality。

5. 把 aggregate prompt 纳入训练

已有 aggregate 的 target 可以用 build_target_mask(labels, component_class_ids) 得到，不需要新增 voxel 标签。建议新增一个 aggregate sampling branch：

Aggregate 类型正样本定义建议采样比例
双侧器官crop/case 中任一 component present 即 positive，target 为 present components 的 union。5% prompt slots
ribs / left ribs / right ribs当前 FOV 内 present rib labels 的 union。5% prompt slots，优先用于 rib-heavy crop
muscle groupsgluteal / iliopsoas component union。2-5% prompt slots

aggregate 不应完全替代 fine-grained class。它的作用是让模型学会从单器官 prompt 迁移到群组 prompt，尤其接近 VoxTell 的 combined-label alternatives。

Vocabulary 是否需要扩增

需要扩增，但扩增边界应受监督标签约束。当前 70 个 class 已通过静态校验，Qwen embedding cache 也有效；问题不是“词表错误”，而是“词表太薄且未被动态采样充分利用”。建议把 vocabulary 从当前 264 prompt records 扩到约 450-650 条高质量 records，而不是追求 VoxTell 9,682 条 rewritten labels 的绝对规模。

建议扩增内容

扩增类型示例方向收益风险控制
每类至少 5 个 validated variants kidney 类加入 renal parenchyma / renal organ / kidney tissue 等；rib 类加入 left 1st rib、left rib number one。 贴近 VoxTell Appendix C 的最低变体数要求，降低 prompt phrasing 敏感性。 laterality prompt 必须自包含结构名；不能只写 left 或 right。
组合概念 variants bilateral kidneys、rib cage、left-sided ribs、hip bones、gluteal muscle group。 补齐 VoxTell combined-label alternatives；当前已有 aggregate schema。 只组合临床上自然成立、且 component mask 可由现有 labels union 得到的概念。
指令式/句子式 prompt segment the left kidney、mask of the urinary bladder、the entire spleen。 让模型更接近 VoxTell 的 free-text prompt 鲁棒性。 句子不能引入未标注属性，如病灶、亚段、边界位置。
同义词与术语规范化 oesophagus/esophagus、gall bladder/gallbladder、clavicle/collarbone。 覆盖用户自然输入和英美拼写差异。 避免高歧义缩写，例如没有上下文时不建议直接训练 GB。
语义边界清理 审查 stomach wall、urinary bladder wall、liver tissue、myocardium and heart 是否与真实 label mask 边界一致。 减少文本与 mask 语义不一致造成的监督噪声。 对于 whole-organ label，优先使用 whole/entire organ 表达；谨慎使用 wall、parenchyma only、tissue only。

不建议扩增的内容

  • 没有 mask 的新 class：例如 aorta、tumor、vessels、left lung/right lung 等，如果 S2I 只有现有 70 labels，不能作为 supervised positive class 加入。
  • 过细 spatial subpart：例如 anterior rib、upper pole kidney、Couinaud segment，如果没有对应 sub-mask，会把完整器官 mask 错当局部描述。
  • 纯语言噪声：拼写错误、缩写、口语表达可以少量加入 robustness validation，但不应在第一轮大量加入训练。

推荐落地顺序

阶段改动成功判据

P1 动态 sampler 实现 set_epoch()、canonical_prob=0.25、class-balanced positive、hard-negative relation table。 同一 case 跨 epoch prompt 不再固定；canonical 比例接近 25%；small class positive slots 提升。
P2 aggregate 训练 让 10 个 aggregate concepts 进入 sampler，并支持 component union target。 aggregate validation 有独立 Dice/FP 指标；fine-grained class Dice 不下降。
P3 vocabulary v1.1 每个 base class 至少 5 条审校 prompt；aggregate 至少 5 条；重建 Qwen embedding cache。 prompt records 约 450-650；静态 validation 无错误；variant robustness val 不低于 canonical val。
P4 patch/ROI sampler 实现 crop 后 present/absent 统计和 85% foreground oversampling。 negative 缺失率接近 0；small structures 在 crop 内出现率显著上升；训练显存可控。
P5 exhaustive validation 保留 sampled val，同时新增 canonical all-class val、variant robustness val、aggregate val。 每个 class 都有可追踪指标，不再只看随机 sampled prompt 的平均值。

实现注意点

  • 扩词必须配套重建 embedding cache：当前 cache 对 prompt records 做 hash 校验，改 label_vocab.json 后旧 cache 会失效。
  • 采样比例要进 config：canonical_prob、aggregate ratio、hard-negative ratio、class-balance cap 都应可复现实验。
  • 验证要拆指标：canonical、variant、aggregate、negative empty、small structures 分开看，否则平均 Dice 会掩盖 prompt 鲁棒性问题。
  • 不要让 aggregate 污染 base class：aggregate 是额外 prompt target，不应把 base label 的 canonical 名称改成群组名。
  • 先小规模 ablation：P1 动态采样和 P3 轻量扩词风险低；P4 patch sampler 改动最大，应单独做 ablation。

最终判断

Body-Tell 当前与 VoxTell 的主要差距不是模型是否能接收文本，而是训练数据分布是否真的逼近 VoxTell：动态 prompt、多样同义表达、局部 patch foreground、局部 absent negative、组合概念和完整验证。Vocabulary 需要扩增，但应作为 sampler 改造后的放大器；如果 sampler 不变，扩增词表的收益会被 deterministic per-case 单 prompt 选择严重限制。
