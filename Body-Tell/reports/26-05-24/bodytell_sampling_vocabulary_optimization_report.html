结论

  1. 第一优先级动态采样基础设施已验收：dataset epoch/iteration RNG 与训练侧 epoch propagation 已通过 SP-B Step 1 review / phase1 Slurm gate。
  2. 第二优先级 patch-level 训练样本对齐已验收：85% foreground patch oversampling、patch 内 present/absent 重算、2 positive + 1 negative sampling 已通过 SP-B Step 2 review / phase1 Slurm gate，并使用 canonical phase1_voxtell_aligned.yaml。
  3. 第三优先级 positive prompt 25/75 策略已验收：Step 3 已实现 25% 默认主名、75% 改写变体，并记录 classes-without-variants fallback metadata。
  4. 第四优先级是 aggregate sampler 支持：当前 next task 为 Step 4；扩 vocabulary 前必须先让训练 sampler 能采到 combined-label concepts。当前 embedding cache 包含 aggregates，但训练索引仍排除了 aggregates。
  5. 第五优先级是 vocabulary 与 embedding 对齐：先扩充并审校 vocabulary，再重建 text embedding cache；这两个步骤有严格先后关系。
  6. 当前数据口径：本报告以当前 S2I-Dataset-70cls/class_presence.json 为准，共 11,070 cases，train/val/test 为 10,070/500/500。

实现顺序

最优顺序是：动态 RNG → patch crop 与 patch 内 2+1 → 25/75 positive prompt → aggregate sampler 支持 → vocabulary expansion → embedding cache 重建。当前动态 RNG、patch crop / patch 内 2+1、25/75 positive prompt 均已验收；下一步是 Step 4，aggregate sampler 支持必须早于 combined-label vocabulary 扩增。

Step 1. 建立训练时动态采样基础设施

执行状态：accepted。证据见 runs/SP-B.Step1/RESULT.html 与 runs/SP-B.Step1/REVIEW.html。

依赖：无。对应 VoxTell：Supplementary A.3 使用 sampled positive/negative prompts，并随机采 synonyms/rephrasings。Body-Tell 当前同一 index 跨 epoch 固定，因此第一步是把随机性从 seed + index 改成训练轮次相关。

  1. 给 HyperBodyPromptDataset 增加 set_epoch(epoch) 或等价机制。
  2. 采样种子使用 base_seed + epoch * large_prime + index；DDP 时确保各 rank 复现但不冲突。
  3. 训练循环每个 epoch 同时调用 DistributedSampler.set_epoch(epoch) 和 dataset 的 epoch setter。
  4. 验收：同一个 case 在两个 epoch 中采到的 prompt 文本可以变化；不再是固定 prompt-conditioned sample。

Step 2. 同步实现 patch crop 与 patch 内 2 positive + 1 negative

执行状态：accepted。证据见 runs/SP-B.Step2/RESULT.html 与 runs/SP-B.Step2/REVIEW.html；review 记录 phase1 Slurm gate job 37959 为 COMPLETED / 0:0。

当前形状现状：Body-Tell 的 S2I-Dataset-70cls/data/*.npz 已在离线 scripts/crop_s2i_dataset.py 中固化为 (128, 128, 256) uint8。yaml 中的 volume_size: [128, 128, 256] 仅在 HyperBodyPromptDataset.__getitem__ 里通过 fit_array_to_shape() 做 center crop/pad，本质是 on-disk safety net；当前 __getitem__ 没有任何随机 crop，训练单位就是整 case 的 128×128×256 体，present/absent 完全沿用 class_presence.json 的整 case 统计。
边界说明：离线 crop_s2i_dataset.py 只是显存与输入尺寸适配，不等价于 VoxTell 训练时的 85% foreground patch oversampling，也不能直接套用论文的 192³ 窗口——Body-Tell 是 anisotropic 的 128×128×256，z 轴是 x/y 的 2 倍。VoxTell-aligned sampling 需要在训练 __getitem__ 内、在 128×128×256 体内按随机分支采 sub-volume 窗口，并在裁出的 patch 上重新计算 present/absent，再基于 patch 内语义采样 2 positive + 1 negative。

依赖：Step 1。对应 VoxTell：输入训练 patch，85% foreground patch oversampling，并在每个训练 patch 上查询 2 条 present structure prompt 和 1 条 absent structure prompt。这个步骤必须整体实现；如果只先裁 patch、但 prompt 仍按 whole-case presence 抽，会产生 target 语义错配。

窗口尺寸决定：采用 patch_size: [128, 128, 128] ——保留 x/y 全幅，只在 z 轴上裁出长度 128 的 sub-volume，等价于"沿身体长轴随机选取一段"。x/y 不丢能避免肝/肺等大结构被切碎，符合 Body-Tell 数据的 anisotropy。该尺寸通过 yaml 配置可改；将 patch_size 临时设为 [128, 128, 256] 即退化为"整 case 即 patch"的方案，便于消融对比。

  1. 新增训练配置（全部写入 canonical phase1_voxtell_aligned.yaml 的 data 段，不硬编码）：
    • volume_size: [128, 128, 256]（保留，on-disk safety net）
    • patch_size: [128, 128, 128]
    • foreground_oversample_prob: 0.85
  2. 在 __getitem__ 中按顺序执行：读取整 128×128×256 case → 采样 crop window → 裁出 occupancy/labels patch → 后续 target mask 和 present/absent 全部基于 patch。
  3. 85% 概率让 crop 窗口覆盖任意 trainable foreground voxel（沿 z 轴随机选含 foreground 的起点）；15% 概率沿 z 轴随机裁；crop 策略和随机分支写入可审计字段。
  4. 对裁出的 patch 重新统计 foreground class voxel counts（不要复用 class_presence.json 的整 case 统计）。
  5. positive classes 从 patch-present classes 中采 2 个。
  6. negative class 从 patch-absent trainable classes 中采 1 个。
  7. negative target 仍为空 mask；positive target 是基于裁后 labels patch 计算的 class mask。
  8. 验收：输出 patch shape 等于配置的 patch_size；foreground/random crop 比例接近 85/15；完整 3 prompt 样本比例高于当前 9,520/10,070；不存在 sampled positive 在 patch 内为空 mask 的样本；将 patch_size 临时设为 [128, 128, 256] 时 sampler 应退化为整 case 行为，以验证回退路径。

Step 3. 对 positive prompt 实现 25/75 主名/改写采样

执行状态：accepted。证据见 runs/SP-B.Step3/RESULT.html 与 runs/SP-B.Step3/REVIEW.html；review 记录 phase1 Slurm gate job 37997 为 COMPLETED / 0:0。Residual risk：train.py 目前依赖 dataset default canonical_prob=0.25，尚未转发非默认配置值。

依赖：Step 1 和 Step 2。对应 VoxTell：positive text prompts 中，默认主名 25%，同义词/改写 75%。这一步只声明 VoxTell 明确写到的 positive prompt 规则，不引入 hard-negative 或额外对比采样。

  1. 每个 prompt record 保留 is_canonical。
  2. 增加配置 canonical_prob: 0.25，只作用于 positive prompt。
  3. 对 positive class：先以 25% 概率选 canonical prompt，否则从非 canonical variants 中随机选；negative prompt 仍保持普通随机采样。
  4. 如果某个 class 暂时没有非 canonical variant，则回退到 canonical，并在审计中标记。
  5. 验收：positive prompt 的 canonical 比例接近 25%，variant 比例接近 75%。

Step 4. 让训练 sampler 支持 aggregate / combined-label concepts

依赖：Step 2。对应 VoxTell：Appendix C 包含 combined-label alternatives。Body-Tell 当前 flatten_prompt_records() 会缓存 aggregates，但 prompt_records_by_class() 使用 include_aggregates=False，因此扩 vocabulary 前必须先让训练 sampler 能采到组合概念。

  1. 把 sampler 的采样单位从 class-only 扩展为 concept：base class concept 的 target_class_ids 为单个 class，aggregate concept 的 target_class_ids 为 component_class_ids。
  2. 对 patch 计算 concept-level present/absent：aggregate 的 union mask 在 patch 内 voxel count ≥ min_voxels 时为 present，否则为 absent。
  3. positive 和 negative 都可以从合法 concept 池中采样；aggregate target mask 是 component class union，negative target 仍为空 mask。
  4. 保留 base class 与 aggregate 的 source_type，避免 combined labels 污染 base class canonical 名称或评估口径。
  5. 验收：当前已存在的 aggregate prompt 能被训练 sampler 采到；aggregate target mask 等于 component masks 的 union；embedding lookup 与 prompt record index 对齐。

Step 5. 按 VoxTell Appendix C 扩 vocabulary

依赖：Step 4。对应 VoxTell：semantic label expansion、cross-dataset harmonization、human expert validation、每个 concept 至少 5 个 variants、combined-label alternatives。Body-Tell 不需要复制 VoxTell 的 1,087 concepts 规模，但应复制构造原则。

  1. 对 70 个 base classes 做 semantic label expansion，每类至少 5 条 validated prompts。
  2. 保留 laterality 自包含原则：所有 left/right prompt 必须包含结构名，不能只写方向词。
  3. 建立 combined-label alternatives：如 kidneys、ribs、left ribs、right ribs、gluteal muscles 等，只使用现有 labels 可以 union 得到的组合。
  4. 做人工/专家审校，移除与真实 mask 语义不一致的 prompt，例如过度暗示 wall-only、parenchyma-only、subpart-only 的表达。
  5. 验收：每个可训练 base class 至少 5 条 prompt；combined labels 有明确 component class ids；静态 vocabulary validation 无错误。

Step 6. 重新预计算 text embeddings

依赖：Step 5。对应 VoxTell：训练时所有 text embeddings 预计算。Body-Tell 已有 cache 机制，所以 vocabulary 变更后必须重建 cache。

  1. 使用同一 instruction：Given an anatomical term query, retrieve the precise anatomical entity and location it represents。
  2. 用 Qwen3-Embedding-4B 对所有 class prompts 和 combined-label prompts 重新编码。
  3. 写入 prompt_embeddings.pt，并更新 vocab hash / prompt records 校验。
  4. 验收：embedding shape 与 prompt records 数量一致，cache hash 与 label_vocab.json 一致。

Vocabulary 扩增边界

Vocabulary 可以扩，但只能按 VoxTell Appendix C 的原则扩：同义/改写、语义统一、combined labels、人工审校。不能引入 S2I 没有 mask 支撑的新监督类别，也不能加入真实 mask 不支持的局部/病灶描述。
允许扩增原因边界
同一 class 的主名、同义词、改写表达 VoxTell 明确使用 synonyms/rephrasings，并要求至少 5 个 variants。 表达必须仍指向同一个完整 label mask。
combined-label alternatives VoxTell Appendix C 明确生成 clinically meaningful groupings。 只能组合现有 class ids，例如左右器官、rib groups、肌肉组。
拼写/术语变体 VoxTell 评估关注 synonyms、rephrasings 和拼写扰动的鲁棒性。 少量加入，且不能产生歧义缩写。
语义边界审校 VoxTell pipeline 包含 harmonization 和 human expert validation。 删除与数据集真实标注范围冲突的 prompt。

明确不纳入本轮

  • 没有 S2I mask 的新监督 class，例如 tumor、vessels、aorta、left lung/right lung。
  • 没有 sub-mask 支撑的局部描述，例如 upper pole kidney、anterior rib、Couinaud segment。
  • 论文没有明确写的采样增强：class-balanced positive、hard-negative、contrastive co-sampling。

实现注意点

  • patch crop 和 patch-level 2+1 必须同一阶段完成：不能先裁 patch 但继续用 whole-case presence 抽 prompt，否则 sampled positive 可能在 patch 内没有 target mask。
  • 离线 128×128×256 不等于 foreground oversampling：crop_s2i_dataset.py 产出的固定 (128,128,256) on-disk 形状只是输入尺寸/显存 profile，不能替代训练时 85/15 foreground/random patch sampler，也不能替代 patch-level present/absent 重算。
  • 不要照抄 VoxTell 的 192³ patch：Body-Tell 是 anisotropic 的 128×128×256，patch_size 选 [128, 128, 128]（保留 x/y 全幅，沿 z 随机裁），通过 yaml 配置；不在代码里硬编码。
  • aggregate sampler 支持必须早于扩词：当前 aggregates 虽然进入 embedding cache，但训练索引排除了 aggregates；直接扩 combined labels 不会改变训练分布。
  • 扩词必须配套重建 embedding cache：当前 cache 对 prompt records 做 hash 校验，改 label_vocab.json 后旧 cache 会失效。
  • 采样比例要进 config：patch_size=[128,128,128]、foreground_oversample_prob=0.85、canonical_prob=0.25、num_positive=2、num_negative=1 应可复现实验。
  • combined labels 不能污染 base class：组合概念是额外 prompt target，不应把 base label 的 canonical 名称改成群组名。
  • patch present/absent 是核心：negative 应基于 patch absent，而不是 whole-case absent。
  • 严格限定范围：本轮不实现论文未明确写到的 class-balanced、hard-negative、contrastive co-sampling。

最终判断

Body-Tell 靠近 VoxTell 的最短路径不是加入更多自定义技巧，而是严格复刻 VoxTell 明确写到的数据构造，并按依赖顺序落地：训练时动态采样、85% foreground patch oversampling、patch 内 2 positive + 1 negative、positive prompt 25/75 主名/改写均已完成；下一步让 sampler 支持 aggregate / combined-label concepts；最后扩充审校 vocabulary 并重建预计算 text embeddings。其他增强项应等这些基础对齐后再单独讨论。
