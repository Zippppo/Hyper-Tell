Body-Tell 对齐 VoxTell v1.1 实施计划

日期：2026-05-24；目标：让 Body-Tell 在结构和权重语义上尽量对齐 VoxTell v1.1，优先支持 VoxTell checkpoint 复用，而不是只做到 tensor shape 能加载。

核验结论

原差异报告中的核心不一致真实存在。 其中最影响 VoxTell 权重复用的是两类问题：一类是参数量和模块拓扑明显不同，例如 encoder/decoder；另一类是参数 shape 一致但 forward 语义不同，例如 transformer decoder。

Encoder backbone 真实存在 Body-Tell 是 5-stage 手写 ConvBlock；VoxTell 是 6-stage ResidualEncoder，features 为 [32,64,128,256,320,320]，blocks 为 [1,3,4,6,6,6]。 当前 encoder 无法复用 VoxTell encoder 权重，也很可能限制性能上限。这个差异不是“轻量配置”，而是 backbone 类型不同。 Body-Tell/body_tell/models/voxtell_body_model.py:155、VoxTell/models-weight/voxtell/voxtell_v1.1/plans.json:20、VoxTell/voxtell/model/voxtell_model.py:131

Decoder 真实存在 Body-Tell 是 4-stage 手写 decoder，GroupNorm + GELU；VoxTell 是 5-stage VoxTellDecoder，使用 StackedConvBlocks 并继承 encoder 的 InstanceNorm3d + LeakyReLU + conv_bias=true。 当前 decoder 无法复用 VoxTell decoder 权重。VoxTell 还总是构造 seg_layers，用于 checkpoint 兼容。 Body-Tell/body_tell/models/voxtell_body_model.py:174、VoxTell/voxtell/model/voxtell_model.py:280
Position / memory 排列 真实存在 Body-Tell 动态生成 D,H,W 顺序的位置编码；VoxTell 使用 init 时固定的 pos_embed buffer，并用 H,W,D 顺序 flatten。 Body-Tell 当前 volume_size=[128,128,256]（见 configs/phase1_voxtell_body.yaml:25，当前数据集输入形状已调整到此尺寸）不是 192³。P6-lite 不加载 VoxTell pos_embed，但 runtime pos 必须和 VoxTell 的 H,W,D memory flatten 语义一致。 Body-Tell/body_tell/models/voxtell_body_model.py:118、VoxTell/voxtell/model/voxtell_model.py:185、:229

目标状态

新增一个 VoxTell-aligned 架构路径，而不是在现有轻量模型上做零散补丁。当前主目标收窄为复用 VoxTell transformer_decoder.* 权重；encoder、decoder、projection 和 full checkpoint 复用只作为后续可选实验，不作为本阶段 P6 目标。

模块目标实现权重复用要求
encoder 使用 dynamic_network_architectures.building_blocks.residual_encoders.ResidualEncoder，参数与 VoxTell v1.1 plan 一致。 P6-lite 不加载 encoder.*；可在后续 full-minus-pos_embed 实验中单独评估。
decoder 移植 VoxTell VoxTellDecoder，保留 5 个 stage、StackedConvBlocks、seg_layers always-built 语义。 P6-lite 不加载 decoder.*；为了后续 full 实验兼容，可保留 self.encoder = encoder 这一共享引用。
transformer_decoder 与 VoxTell 的 TransformerDecoderLayer 行为一致：默认 relu，pre-norm 不执行 self-attention。 P6-lite 唯一必需加载目标：同名加载 VoxTell transformer_decoder.* 权重；保留 self-attention 参数，即使 active path 不使用。
project_bottleneck_embed 使用 VoxTell 命名，替代当前 project_visual_embed；结构仍为 320 -> 2048 -> 2048。 P6-lite 默认不加载；如要做 transformer+projectors 对照，可额外加入显式 prefix。
project_text_embed 保持 2560 -> 2048 -> 2048。 P6-lite 默认不加载；如要做 transformer+projectors 对照，可额外加入显式 prefix。
project_to_decoder_channels + num_heads 使用 VoxTell 命名，替代当前 mask_projections；5 级；输出维度 = encoder 各级 channels × num_heads。当前 P6-lite 不复用这部分权重，因此 num_heads 不需要为了权重加载而改变。 P6-lite 明确不加载 project_to_decoder_channels.* 和 decoder mask-fusion 权重；如后续做 full 实验，再以实际 v1.1 checkpoint audit 结果决定是否加载。
pos_embed 只保留 runtime 位置编码要求：按当前 selected feature 的真实 H,W,D 生成，与 VoxTell 的 memory flatten 顺序保持一致。 P6-lite 不加载 checkpoint 中的 pos_embed，也不要求实现 fixed_voxtell_192。

实施阶段

P0：建立兼容性基线

  1. （2026-05-24 修订）transformer_decoder.* prefix 审计：新增或更新 Body-Tell/scripts/audit_voxtell_alignment.py（如果与已有 scripts/parameter_audit.py 重合，应共享 utilities）。脚本必须读取 VoxTell v1.1 checkpoint，并检查 transformer_decoder.* 在 Body-Tell aligned transformer 中 key 集合和 shape 是否完全一致。
  2. 实例化 Body-Tell aligned model 和 VoxTell v1.1 model，输出总参数、模块参数、transformer_decoder.* state dict key/shape 覆盖率。num_heads 与 projection shape 只作为信息项记录，不再阻塞 P6-lite。
  3. 新增 transformer golden test Body-Tell/tests/test_voxtell_transformer_compat.py：同一小尺寸 state dict、相同输入、dropout=0 时，Body-Tell aligned transformer 输出必须与 VoxTell transformer 完全一致或在 1e-6 内。
  4. 新增 weight key audit：读取 VoxTell/models-weight/voxtell/voxtell_v1.1/fold_0/checkpoint_final.pth（已确认文件存在）的 network_weights，只检查 transformer_decoder.* 目标 key 是否全部可加载。

验收：当前轻量模型仍可跑旧测试；新增审计能明确报告 transformer_decoder.* 会加载的 tensor 数、参数数和任何 missing/unexpected/shape mismatch。

P1：先修 transformer decoder 语义

  1. 把 Body-Tell/body_tell/models/transformer.py 的 active forward_pre 改为 VoxTell 版本：从 norm2 -> cross_attn -> norm3 -> FFN 开始，不执行 self-attention。
  2. 构造层时去掉 activation="gelu"，或显式设为 relu，与 VoxTell 默认一致。
  3. 保留 self_attn、norm1 等参数名和 shape，保证 checkpoint key 不变。

验收：Body-Tell aligned transformer 与 VoxTell transformer 小模型 parity test 通过；加载 VoxTell transformer_decoder.* 后无 missing/unexpected/shape mismatch。

P2（可选，非 P6-lite 必需）：替换 encoder 为 VoxTell residual encoder

  1. 新增配置文件 Body-Tell/configs/phase1_voxtell_aligned.yaml，设置 encoder_channels: [32,64,128,256,320,320]、n_blocks_per_stage: [1,3,4,6,6,6]、decoder_layer: 4、num_maskformer_stages: 5。
  2. 在模型配置中加入 architecture: voxtell_v1_1 或 backbone: residual_encoder，让旧轻量模型和 aligned 模型可并存。
  3. 使用 VoxTell 相同构造参数：conv_bias=true、InstanceNorm3d(eps=1e-5, affine=True)、LeakyReLU(inplace=True)、BasicBlockD、return_skips=True、disable_default_stem=False。
  4. 把依赖写入 Body-Tell 环境声明：dynamic-network-architectures>=0.4.1,<0.5 和 einops。若采用 VoxTell 原始位置编码实现，再加入 positional-encodings。

验收：encoder 参数约 90.30M；输出 6 个 skip；encoder.output_channels 和 encoder.strides 与 VoxTell 一致。encoder.* 权重复用不属于 P6-lite，后续 full 实验再单独审计。

P3（可选，非 P6-lite 必需）：替换 decoder 为 VoxTellDecoder

  1. 移植 VoxTellDecoder 到 Body-Tell 模型代码或独立模块，优先保持源码结构、字段名和 state dict key 兼容。
  2. 使用 n_conv_per_stage_decoder=[1,1,1,1,1]，并继承 encoder 的 conv/norm/nonlin/dropout/bias。
  3. 即使 deep_supervision=false，也总是构造 5 个 seg_layers，与 VoxTell checkpoint 兼容。
  4. 移除 Body-Tell 当前 decoder 中 upconv 后的自动 F.interpolate fallback；如果输入尺寸不能通过转置卷积对齐，应在测试中显式暴露，或在数据裁剪 policy 中解决。

验收：decoder 本体参数约 12.18M；5 个 stage / 5 个 transposed conv / 5 个 seg layer。decoder.* 权重复用不属于 P6-lite，后续 full 实验再单独审计。统计 decoder 参数时需扣除共享 encoder，避免误读为 102.47M。

P4（可选结构对齐）：对齐 projection、mask head 和 forward 排列

P4 不再作为 P6-lite 前置：当前只加载 transformer_decoder.*，因此 num_heads、project_to_decoder_channels.* 和 decoder mask-fusion 权重不参与 P6-lite。默认保留 Body-Tell 当前 num_heads=32；如果后续重启 full checkpoint 实验，再单独做完整 projection/mask-fusion audit。
  1. （2026-05-24 修订）num_heads 不为 P6-lite 改动：保留当前配置；只需保证 forward 中所有依赖 num_heads 的 reshape（如 view(B, num_heads, C//num_heads)）和 decoder stage 内 input_features + num_heads 的通道拼接在当前配置下有单测覆盖。
  2. 把当前 project_visual_embed 改为 VoxTell 命名 project_bottleneck_embed；把 mask_projections 改为 project_to_decoder_channels。
  3. encoder 变为 6-stage 后，projection stage 自然恢复为 5 级，不再用 len(encoder_channels)-1 隐式截断配置。
  4. forward 中使用 VoxTell 的 memory 排列：b c d h w -> b h w d c -> (h w d) b c。
  5. decoder 输出仍保持 Body-Tell 训练和推理需要的 (B, N, D, H, W) 语义；如果内部沿用 VoxTell 注释中的 h,w,d 命名，需要在单测里用非立方输入验证轴没有被误换。

验收：5 个 projection 输出维度与当前配置一致；非立方输入的输出 shape 与 target mask 完全一致；mask fusion 模块在当前 num_heads 下有显式单测。P6-lite 不要求这些权重可从 VoxTell checkpoint 加载。

P5/P6-lite：加载 transformer decoder 权重

P5 已并入 P6-lite：位置编码不再作为独立阶段；它只是加载并复用 transformer_decoder.* 前必须满足的 forward 输入契约。
  1. 位置编码契约：不加载 VoxTell checkpoint 中的 pos_embed，也不实现 fixed_voxtell_192。forward 时按 selected feature 的实时 H,W,D 生成 deterministic 3D positional encoding，并保证 pos 与 memory 的 token 数、batch 维、dtype、device 完全一致。
  2. 排列契约：memory 和 pos 必须使用 VoxTell 一致的 flatten 语义：b c d h w -> b h w d c -> (h w d) b c。这是 transformer 权重可迁移的必要条件，否则 shape 可加载但 cross-attention 空间语义会错。
  3. 加载入口：新增训练参数 --init-transformer-from-voxtell VoxTell/models-weight/voxtell/voxtell_v1.1，在 optimizer 构造前加载。
  4. prefix loader：读取 fold_0/checkpoint_final.pth 的 network_weights，只筛选 transformer_decoder.*。
  5. 把筛选出的 key 映射到 model.transformer_decoder.state_dict()，要求 key 集合完全一致、shape 完全一致，然后对 model.transformer_decoder.load_state_dict(..., strict=True)。
  6. 不加载 encoder.*、decoder.*、project_bottleneck_embed.*、project_text_embed.*、project_to_decoder_channels.* 或 pos_embed；不需要 full-loader 白名单。
  7. 日志记录 loaded tensor 数、loaded 参数数、checkpoint 路径、prefix 和任何 missing/unexpected/shape mismatch，并写入训练 checkpoint 的 config.init_transformer_from_voxtell。
  8. 不要把旧 Body-Tell checkpoint 与 aligned 模型静默混用。旧 checkpoint 应只用于旧架构；aligned checkpoint 应带 architecture: voxtell_v1_1。

验收：在 CPU 上能完成 transformer_decoder.* prefix 加载审计；无 missing/unexpected/shape mismatch；非立方输入下 memory/pos 契约检查通过；在 GPU 上能完成至少 1 个 batch 的 forward/loss/backward smoke。
