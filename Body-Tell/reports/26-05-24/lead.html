SP-C 执行边界

SP-C 拆成两类执行，避免误把训练基础设施全部拖到结构和数据完成之后。

  • 可以现在做：Task 0 profile 骨架、Task 1 fixed-update loop、Task 2 SGD profile、Task 3 poly schedule、Task 4 checkpoint/resume。
  • 必须等前置完成：Task 5 deep supervision 等 SP-A P3；Task 6 的最终评估口径等 SP-B patch-level sampler 和 vocabulary/cache；长训通过等端到端 smoke 通过。
明确不做项（2026-05-24 决议，不得在 SP-C 中再次提案）：
  1. 不严格对齐论文 2,000 × 250 = 500k step 的训练预算。2026-05-24 锁定实际预算：batch_size=2 per GPU × 2 GPUs（全局 4）、epochs=75、iterations_per_epoch=250 → total_updates=18,750 step（约论文 500k 的 3.75%）。paper_aligned profile 中必须以独立字段 compute_budget 记录上述数字，并在训练日志显式注明 "训练预算 = 18,750 step（bz=2×2, ep=75），非论文 500k"，避免误读为复刻。
  2. 不切换到 nnU-Net nnUNetTrainer 框架。VoxTell 训练侧代码未开源，"切到 nnU-Net"等于自己重写 trainer，收益不及成本。继续使用当前自定义 PyTorch loop 作为 SP-C 改造的载体，只在 loop 内补齐 iteration 语义、SGD、poly schedule、scheduler state checkpoint 等机制级对齐；不引入 nnU-Net 框架依赖、不模仿其 on_train_epoch_start 等钩子结构。

Baseline 评估闸门（2026-05-24 新增）

在进入"paper-aligned 长训"之前，必须先通过下列 baseline 闸门，避免结构对齐与权重加载在 silent regression 下被误判为收益：
  1. 结构闸门（SP-A P0 完成后）：audit_voxtell_alignment.py 报告 transformer parity test 通过（输出差异 < 1e-6，dropout=0）；num_heads 与每级 projection 实际 shape 已显式打印。
  2. 权重加载闸门（SP-A P6 完成后）：strict=False 加载 VoxTell checkpoint_final.pth 时，unexpected_keys 集合等于 P4 决议产出的显式白名单——路径 A 为 {"pos_embed"}，路径 B 为 {"pos_embed"} ∪ {project_to_decoder_channels.*, mask_fusion.*}；missing_keys 为空；任何 shape mismatch 必须抛错。
  3. 数值闸门（aligned random 训练 2 epoch smoke 后）：val foreground Dice 不得低于当前 baseline（约 0.3257）的 70%，否则回退 SP-A / SP-B 已落地改动，不进入长训。该阈值仅用于"明显回归"识别，不是性能目标。
  4. 评估闸门（SP-C Task 6 完成后）：known concepts / prompt variants / negative empty-mask / held-out 四组评估全部能输出指标（即使分数不理想），且 prompt records / vocab hash 与 checkpoint 一致。
