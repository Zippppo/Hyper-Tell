Body-Tell / VoxTell 实现差异精简报告

日期：2026-05-24；范围： Body-Tell/body_tell/models/*、 Body-Tell/configs/phase1_voxtell_body.yaml、 VoxTell/voxtell/model/*、 VoxTell/models-weight/voxtell/voxtell_v1.1/plans.json。

结论

当前 Body-Tell 不是“VoxTell 换数据集”的近似复刻，而是保留了 prompt-decoder 的参数规模，同时把 image encoder / U-Net decoder 重写成了更轻的自定义版本。

已有参数报告的结论成立：总差距约 111.05M，其中 encoder 差距约 81.80M。但更关键的是，部分参数量一致的模块也存在行为不等价。

Body-Tell 总参数328.98M
VoxTell 总参数440.03M
encoder 差距81.80M
最高优先级风险Transformer 语义
  1. transformer_decoder 参数量一致，但 active pre-norm forward 不一致。
  2. Body-Tell encoder 是 5-stage 普通 ConvBlock；VoxTell 是 6-stage nnU-Net residual encoder。
  3. Body-Tell decoder 是手写 _VoxTellBodyDecoder；VoxTell decoder 依赖 dynamic_network_architectures 的 encoder 元信息与 StackedConvBlocks。
  4. Body-Tell 实际 mask projection 只有 4 级；VoxTell 是 5 级。

1. Encoder：不是同一类 backbone

Body-Tell 配置为 encoder_channels: [32, 64, 128, 256, 320]，共 5 个 stage。代码中 _BodyEncoder 每个 stage 只堆一个 _ConvBlock；每个 _ConvBlock 是两层 Conv3d + GroupNorm + GELU，无 residual skip，卷积 bias=False。

VoxTell v1.1 plan 是 6-stage residual encoder：[32, 64, 128, 256, 320, 320]，n_blocks_per_stage=[1, 3, 4, 6, 6, 6]，InstanceNorm3d(affine=True)，LeakyReLU，conv_bias=true。模型代码直接构造 ResidualEncoder(..., return_skips=True, disable_default_stem=False)。

encoder8.49M90.30M不同 backbone，不是小配置差异。

证据：Body-Tell/body_tell/models/voxtell_body_model.py:46、:155、:254；VoxTell/models-weight/voxtell/voxtell_v1.1/plans.json:17、:20、:94、:109；VoxTell/voxtell/model/voxtell_model.py:131。

2. Transformer decoder：shape 对齐但行为不对齐

两边都是 6 层、d_model=2048、8 heads，参数量同为 251.86M。但 Body-Tell 构造时显式 activation="gelu"，VoxTell 构造时使用 TransformerDecoderLayer 默认 activation="relu"。

路径Body-TellVoxTell
active forward_pre norm1 -> self_attn -> norm2 -> cross_attn -> norm3 -> FFN norm2 -> cross_attn -> norm3 -> FFN
self_attn 权重 active path 使用 active path 未使用

同权重小实验确认了语义差异：state_dict 完全相同、dropout=0、输入相同，Body-Tell 与 VoxTell pre-norm 输出最大绝对差约 0.7087，attention 最大差约 0.0200。这说明“shape 可加载”不等于“权重语义等价”。

证据：Body-Tell/body_tell/models/voxtell_body_model.py:69、:74；Body-Tell/body_tell/models/transformer.py:137；VoxTell/voxtell/model/voxtell_model.py:191；VoxTell/voxtell/model/transformer.py:130、:224。

3. Mask projections：配置写 5，运行实际只有 4

Body-Tell 配置写 num_maskformer_stages: 5，但代码用 min(num_maskformer_stages, len(encoder_channels)-1)，因此实际只有 4 个 projection，输出维度为 [32, 2048, 4096, 8192]。

VoxTell 使用 DECODER_CONFIGS 前 5 级，输出维度为 [32, 2048, 4096, 8192, 10240]。

模块Body-TellVoxTell差异
mask projection46.23M71.40M+25.18M

这一级缺失不是单纯参数少，而是 decoder 少了一个 320-channel fusion stage。证据：Body-Tell/configs/phase1_voxtell_body.yaml:14；Body-Tell/body_tell/models/voxtell_body_model.py:83、:88；VoxTell/voxtell/model/voxtell_model.py:38、:170。

4. Decoder：手写实现与 VoxTellDecoder 不等价

Body-Tell decoder 是 _VoxTellBodyDecoder，共 4 个 upconv/stage；每个融合 block 复用 _ConvBlock，即两层 Conv3d + GroupNorm + GELU。在 deep_supervision=False 时，中间 seg_layers 是 Identity。代码还会在 upconv 后 size 不一致时 F.interpolate 到 skip shape。

VoxTell decoder 是 VoxTellDecoder，共 5 个 upconv/stage；每个 stage 用 StackedConvBlocks(n_conv_per_stage=1)，继承 encoder 的 Conv3d + InstanceNorm3d + LeakyReLU + conv_bias=true。即使 deep_supervision=False，也总是构造 seg_layers 以保持权重加载兼容。

模块Body-TellVoxTell判断
decoder own params8.11M12.18MVoxTell 多一级 320->320 stage，且 block/norm/nonlin/bias 都不同。

注意：VoxTell 的 VoxTellDecoder 内部保存了 self.encoder = encoder，所以直接统计 model.decoder.parameters() 会把共享 encoder 也算进去；比较 decoder 本体时需要扣除共享 encoder 参数。证据：Body-Tell/body_tell/models/voxtell_body_model.py:174、:192、:202、:204、:225；VoxTell/voxtell/model/voxtell_model.py:351、:361、:384、:393。

5. Projection / position / memory 排列

视觉投影与文本投影的参数结构基本一致：visual 为 320 -> 2048 -> 2048，text 为 2560 -> 2048 -> 2048。

  • Body-Tell 选 decoder_layer=-1，即 5-stage encoder 的最后一级；VoxTell 推理入口选 decoder_layer=4，即 6-stage encoder 的倒数第二级。二者都为 320 channel，但空间层级语义不同。
  • Body-Tell 每次 forward 动态生成 sinusoidal_position_encoding_3d(depth,height,width)；VoxTell 在 init 时注册固定 pos_embed buffer，shape 来自 plan 中 decoder layer 4 的 (12,12,12)。
  • Body-Tell flatten 顺序是 D,H,W；VoxTell 代码 reshape 为 H,W,D 后 flatten。立方体 patch 上影响不明显，但 Body-Tell 当前输入体素是 [164,129,256]，非立方体场景下应显式对齐。

迁移建议

  1. 先对齐 transformer decoder：relu、VoxTell-compatible forward_pre、跳过或重建 pos_embed buffer。否则 decoder 权重 shape 虽能加载，但语义不等价。
  2. 将 Body-Tell encoder 改为 dynamic_network_architectures 的 ResidualEncoder，使用 6-stage [32,64,128,256,320,320]、[1,3,4,6,6,6]、InstanceNorm3d、LeakyReLU、conv_bias=true。
  3. 将 decoder 改为 VoxTellDecoder 语义：5 个 decoder stage、StackedConvBlocks(n_conv_per_stage=1)、继承 encoder 的 norm/nonlin/bias、始终构造 seg_layers。
  4. encoder 改为 6-stage 后，mask projection 自然恢复为 5 级，补上 320 * 32 = 10240 的最高级 projection。
  5. 对权重复用实验单独建兼容配置，避免把当前 Body-Tell checkpoint 与 VoxTell-compatible transformer 直接混用。

核验说明

本次没有修改模型代码。参数核验使用当前环境实例化 Body-Tell，并用兼容 stub 绕过缺失的 positional_encodings 依赖来实例化 VoxTell 架构；该 stub 只影响固定 positional buffer 的具体值，不影响参数结构、stage 数和 forward 源码判断。
