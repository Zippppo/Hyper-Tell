总判断

当前观察到的“性能更好且过拟合消失”，最合理的解释不是单点模型结构改动，而是三类变化叠加：

  1. 训练样本分布从固定 whole-case 监督变成 VoxTell-style 动态 patch/prompt 监督。这是最强归因。它直接减少固定 case、固定 class、固定文本的记忆路径，并让每个 epoch 都看到不同的局部 present/absent 组合。
  2. 语言与概念空间扩展。每个可训练基础类有 5 条 prompt，加入 aggregate union concept，并重建 Qwen embedding cache，使模型不能只拟合单一 canonical 名称。
  3. VoxTell-aligned 架构与训练闭环修正。Residual encoder、VoxTell decoder、Transformer prefix loader、动态 HWD position encoding、deep-supervision 兼容、DDP/global metric 修正等提高了可训练性、迁移先验和评估可信度。

证据边界：5 月 26/27 的 aligned 曲线最符合“过拟合消失”，但这些 run 早于 SP-C T4 的 DDP 全局指标修复；且 0526S2I-aligned128/best.pt 后来被 deep-supervision 试跑覆盖。因此本报告把训练曲线作为强线索，不把它写成完全闭环的最终 A/B 证明。

改动点与作用机制

改动点 原问题 当前落地位置 为什么能提升/缓解过拟合 归因强度
epoch-aware 动态采样 同一个 dataset index 跨 epoch 采到固定 prompt/class 组合，模型容易把 case 与固定文本监督绑定。 HyperBodyPromptDataset.set_epoch()；train._set_training_epoch()。 同一 case 每个 epoch 看到不同 prompt、positive/negative concept 和 crop，形成语义级数据增强。抽样审计中 500/500 个 index 的 prompt set 在 epoch 2 全部变化。 高
85% foreground patch oversampling 旧 whole-case 训练容易被背景和大结构支配；小器官/稀疏结构进入有效梯度的概率低。 data.patch_size=[128,128,128]；foreground_oversample_prob=0.85；dataset._sample_patch()。 训练从 128x128x256 的整 case 变为沿 z 轴随机局部 patch；85% 分支强制覆盖 trainable foreground，提高目标 voxel 出现率和梯度密度。它也使同一病例产生更多局部上下文，抑制全局模板记忆。 高
patch 内 present/absent 重算 如果裁 patch 但继续用 whole-case presence，positive 可能在 patch 内为空，negative 也可能实际存在，形成错训。 dataset._patch_case_record() 后再调用 sample_prompts_for_case()。 positive 必须来自当前 patch 内 present concept，negative 来自当前 patch 内 absent concept。抽样审计中空 positive mask 为 0，说明监督语义和 target mask 对齐。 高
2 positive + 1 negative prompt 缺少 negative prompt 时，模型只学习“看到文本就输出器官”，容易在空 prompt 上幻觉。 num_positive=2、num_negative=1；sample_prompts_for_case()。 每个样本同时学习目标分割和空目标抑制，负 prompt FP 维持在 1e-4 量级。它让 prompt-conditioned segmentation 学到“文本不存在时输出空 mask”的边界。 高
positive prompt 25/75 主名/改写 只用 canonical 名称时，文本编码空间过窄，模型容易把一类器官绑定到单个词。 canonical_prob=0.25；_sample_positive_prompt_record()。 大多数 positive 使用同义/改写表达，使模型必须学习文本语义邻域而不是固定字符串。当前抽样 canonical 比例约 22%。 高
vocabulary expansion + Qwen cache 重建 词表变体不足、cache 与 prompt records 错位都会造成训练信号窄或错误。 configs/label_vocab.json；artifacts/text_embeddings/prompt_embeddings.pt；validate_prompt_embedding_cache_data()。 当前 69 个 trainable class 每类 5 prompts，14 个 aggregates，总 418 prompt records；embedding shape 为 (418, 2560)，hash/records/finite 校验通过。语义覆盖变宽，embedding 行与 target 保持一致。 高
aggregate / combined-label concept sampler 训练只看单 class 时，模型对“kidneys/ribs/gluteal muscles”等组合 prompt 没有直接监督。 prompt_records_by_concept()；aggregate component_class_ids union target。 组合概念给出 group-level union mask，既扩展 prompt 语义，也对多个相关器官施加一致监督。抽样中约 45% 样本含 aggregate prompt。 中高
VoxTell-aligned residual encoder 旧 encoder 与 VoxTell profile 不一致，结构较浅，不能承接后续 decoder/mask-head 设计。 backbone=residual_encoder；channels [32,64,128,256,320,320]；blocks [1,3,4,6,6,6]。 残差路径和更深 stage 提高 3D 特征表达与优化稳定性。但它也增加参数量，因此单独看不一定减少过拟合；必须和动态采样一起解释。 中
Transformer decoder 语义对齐 + prefix loader 如果 forward_pre、activation、key shape 与 VoxTell 不一致，加载同名权重也可能是语义错配。 TransformerDecoderLayer(activation="relu", normalize_before=True)；load_voxtell_transformer_decoder_prefix()。 严格加载 110 个 transformer_decoder.* tensor、约 251.86M 参数，把已训练的 text-image fusion 先验迁移过来，减少从零学习 prompt fusion 的样本需求。 中
动态 HWD position encoding VoxTell 固定 pos_embed 不适合 Body-Tell 128x128x256/patch 非立方输入。 B,C,D,H,W -> B,H,W,D,C -> H*W*D,B,C；runtime sinusoidal 3D position。 避免位置编码 shape 错配或 silent interpolation，让 prompt-image memory 排列和 VoxTell 语义一致。主要提升稳定性和可迁移性。 中
训练/评估闭环修复 deep supervision bundle 被 metrics 当 tensor；negative-only batch 可能污染 Dice；DDP rank-local 指标会误选 best。 _primary_logits_for_metrics()；foreground Dice sum/count；_reduce_ddp_metric_totals()；train/inference build_model 对齐。 这些改动主要修正“训练是否真的按预期跑”和“指标是否可信”，不一定直接提高模型能力，但会防止错训、错评和错误 checkpoint 选择。 中

为什么不是“模型变大所以更好”

参数量变化

旧 run 约 328.98M 参数；aligned run 约 440.03M 参数。更大的模型通常会增加记忆训练集的能力，因此“gap 变小”不能由参数增加自然解释。

曲线形态

旧 whole-case run 可以达到更高 train Dice，但 val 不同步提升；aligned run 的 train Dice 较低、val 接近 train。这更像采样/语义增强带来的正则化，而不是容量驱动的训练集拟合。

贡献排序

排序因素判断理由
1 patch-level dynamic 2+1 sampling 最可能主因 直接改变训练分布和监督正确性；抽样证据完整；与过拟合 gap 缩小高度一致。
2 prompt variant / vocabulary / aggregate 强正则化来源 文本空间从少量固定名称扩到 418 records，并进入每 epoch 随机采样。
3 VoxTell-aligned architecture + transformer prefix 重要但非单独充分 提升 prompt-image fusion 先验和结构兼容性；但 encoder/decoder/projectors 未加载 VoxTell 权重。
4 训练/评估契约修正 可信度和稳定性因素 防止 deep supervision、NaN Dice、DDP rank-local best 等静默错误；对“真实性能”判断很关键。
5 更多 S2I train cases 辅助因素 旧审计中 train 9779，当前 active split 10070；数据增加有帮助，但幅度不足以单独解释 gap 从 0.18 降到 0.006。

风险与后续验证

  • 需要 post-T4 长训复核：重新跑一条不覆盖 checkpoint 的 aligned 长训，确保 DDP 全局 reduction 后仍复现 gap 小、val Dice 高的曲线。
  • aggregate 监督需要消融：当前 vocabulary 中 aggregate 的 train_as_positive/eval_as_foreground 为 false，但 sampler 会把组件全可训练的 aggregate 纳入训练 concept。这可能有益，也可能和单器官边界产生冲突，建议做 no-aggregate ablation。
  • deep supervision 不能归因给 5/26 曲线：5/26 aligned run 的 config 里 deep_supervision=false；当前 config 已为 true，仍需独立长训证明它是否继续提升。
  • prefix loader 边界要保持清晰：P6-lite 只加载 transformer_decoder.*，不应把收益表述成完整 VoxTell checkpoint 迁移。
  • best checkpoint 选择可加入负 prompt 约束：当前 best 主要看 foreground_mean_dice，建议至少记录并门控 negative_fp_rate，避免选中空 prompt 幻觉偏高的模型。

结论

最有解释力的因果链是：动态 patch/prompt sampling 修正了训练分布和监督语义，扩词与 aggregate 提供了语言层正则化，VoxTell-aligned 架构和 transformer prefix 提供了更好的融合先验，再由训练/评估闭环修正确保指标和 checkpoint 选择不被实现细节污染。

因此，当前性能提升和过拟合减轻更像是“输入分布、语义监督、迁移先验、评估契约”四者对齐后的结果，而不是某一个网络层替换单独造成的结果。
