Body-Tell 旧 Dataset 删除指导报告

日期 2026-05-24 · 范围仅限 Body-Tell/ · 目标: 删除旧 Body-Tell/Dataset 数据和旧 Dataset fallback 逻辑, 以 S2I-Dataset-70cls 作为唯一数据源。

结论

当前 Phase 1 S2I 迁移已经具备删除旧 Body-Tell/Dataset 的条件。建议执行顺序是: 先清理代码中的旧 fallback 和默认路径, 跑完验证, 再删除 Body-Tell/Dataset/。 这样删除后所有入口都会明确依赖 S2I dataset package, 不会再静默回退到旧数据布局。

架构决策

决策项最终规则理由
CANONICAL 采用 dataset-local package 规则: Body-Tell/S2I-Dataset-70cls/ 内同时保存 data/, dataset_info.json, dataset_split.json, class_presence.json。 class_presence.json 是针对具体数据集扫描得到的 case-level presence/statistics, 不是全局类别定义。它应与对应 dataset 放在一起, 避免同一功能 artifact 同时出现在两个 canonical-looking 位置。
NO COPY 不把 S2I-Dataset-70cls/class_presence.json 复制或替换到 Body-Tell/artifacts/data_stats/class_presence.json。 复制会产生两个副本, 后续容易出现 stale / 不知道哪个才是 active source 的问题。 本次迁移目标是消除旧 Dataset 语义, 不是保留旧 artifacts 路径作为兼容层。
SINGLE SOURCE 当前唯一有效 presence 文件是 Body-Tell/S2I-Dataset-70cls/class_presence.json。 Body-Tell/artifacts/data_stats/class_presence.json 作为旧 Dataset 派生产物删除。 两个文件类别身份一致, 但 case 数、split、shape、每类 case_count / total_voxels 等统计不同。 S2I 训练必须使用 S2I 自己的 presence。

代码清理清单

文件旧逻辑推荐改法
body_tell/data/dataset.py 无显式参数时默认读取 Dataset/dataset_split.json, Dataset/voxel_data, artifacts/data_stats/class_presence.json。 删除旧 Dataset fallback。建议让 voxel_dir, split_path, presence_path 必须由 config 显式传入, 或将默认值改为 dataset-local S2I: S2I-Dataset-70cls/data, S2I-Dataset-70cls/dataset_split.json, S2I-Dataset-70cls/class_presence.json。
inference.py resolve_case_paths() 在缺少 data config 时仍默认 Dataset/voxel_data 和 Dataset/dataset_split.json。 删除旧 fallback。推理入口应优先使用 checkpoint/config 内的 data.voxel_dir 和 data.split_path; 如果缺失, 要么使用 dataset-local S2I 默认路径, 要么直接报错。
scripts/validate_phase0.py 默认参数指向旧 Dataset 和旧 presence。 默认参数改为 S2I: S2I-Dataset-70cls/dataset_info.json, S2I-Dataset-70cls/data, S2I-Dataset-70cls/class_presence.json。
scripts/scan_class_presence.py 默认参数指向旧 Dataset 和旧 output。 默认输入 / 输出改为 S2I dataset package, 输出到 S2I-Dataset-70cls/class_presence.json。保留命令行参数覆盖能力, 但默认行为不再依赖旧 Dataset 或 artifacts/data_stats。
scripts/build_label_vocab.py metadata 和 help text 写死 Body-Tell/Dataset/dataset_info.json / Dataset/voxel_data。 改成 S2I dataset_info, 或改成由 --dataset-info 决定的动态来源字符串。 注意修改 configs/label_vocab.json 会改变 vocab hash, 需要同步重建 prompt_embeddings.pt; 如果不计划重建 cache, 不要改会影响 hash 的 vocab 内容。
tests/ fixture 中仍使用旧 Dataset 字符串模拟最小旧布局。 可以保留测试 fixture 的临时目录结构, 但建议新增删除后回归测试: 在没有 Body-Tell/Dataset 的情况下构建 S2I dataset、resolve inference case、运行 validate。

推荐执行顺序

  1. 修改 runtime 默认路径: dataset.py, inference.py, validate_phase0.py, scan_class_presence.py 不再指向旧 Dataset。
  2. 保留 label_vocab.json 内容不动, 除非准备同步重建 embedding cache。当前 cache 与 vocab hash 匹配, 不应无意改动。
  3. 新增或调整测试, 覆盖 “Body-Tell/Dataset 不存在时 S2I 仍可运行”。
  4. 跑验证命令, 确认无旧 Dataset 运行依赖。
  5. 删除 Body-Tell/Dataset/。
  6. 删除旧 Dataset presence 派生产物 Body-Tell/artifacts/data_stats/class_presence.json, 不做 S2I 文件复制替换。
  7. 再次跑同一组验证命令, 确认删除后仍通过。
  8. 可选删除旧演示输出: Body-Tell/outputs/inference_demo/。

风险与注意事项

  • configs/label_vocab.json 当前仍含旧来源字符串。这是 metadata/notes, 不是 runtime 路径依赖。直接修改它会改变 vocab hash, 导致现有 prompt_embeddings.pt 严格校验失败, 除非同步重建 embedding cache。
  • 不要把 S2I 的 class_presence.json 复制到 artifacts/data_stats/。本项目最终采用 dataset-local package, presence 文件只保留在对应数据集目录下。
  • 删除 Body-Tell/Dataset 后, 任何未显式传 S2I config 且仍依赖旧默认路径的脚本都会失败。这正是本次代码清理要提前消除的风险。
  • 旧报告 HTML 中出现旧 Dataset 路径属于历史记录, 不需要作为 runtime 阻塞项处理。
  • 如果需要保留删除前取证, 可先记录 du -sh Body-Tell/Dataset 和 find Body-Tell/Dataset -type f | wc -l 输出。

最终建议

可以删除旧 Body-Tell/Dataset, 但不建议直接从物理删除开始。 最稳妥的完成定义是: 先提交代码清理, 让所有默认入口都以 Body-Tell/S2I-Dataset-70cls 作为唯一 dataset package; 然后删除旧目录和旧 artifacts/data_stats/class_presence.json; 最后用同一组验证命令确认删除后仍可训练、推理、可视化和验证。
