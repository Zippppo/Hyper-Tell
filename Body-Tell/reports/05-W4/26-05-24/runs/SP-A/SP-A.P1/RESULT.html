SP-A.P1 - Align transformer decoder semantics

Date: 2026-05-24. Scope: transformer decoder behavior parity using SP-A.P0 golden-test material.

Summary

Aligned Body-Tell's active pre-norm transformer decoder path with VoxTell v1.1: the path now starts from norm2 cross-attention and then runs norm3 FFN, without executing self-attention. The self-attention module and norm1 parameters remain present so checkpoint keys and shapes are preserved.

Changed Body-Tell model construction to use activation="relu" for the transformer decoder layer, matching VoxTell's default decoder activation.

Files Changed

  • Body-Tell/body_tell/models/transformer.py - removed self-attention execution from TransformerDecoderLayer.forward_pre.

  • Body-Tell/tests/test_voxtell_transformer_compat.py - converted the SP-A.P0 golden parity test from expected-fail to active pass/fail validation.
  • Body-Tell/reports/26-05-24/runs/SP-A.P1/RESULT.html - this result report.
