# HyperBody Voxel Dataset 数据集说明

## 项目概述

本数据集用于**输入部分人体表点云，预测全人体体素及类别**的深度学习任务。数据集包含人体表面点云作为输入，以及对应的完整人体三维体素网格和解剖结构类别标签作为输出。

## 数据集概览

- **数据集路径**: `Dataset/voxel_data/`
- **文件总数**: 10,779 个样本
- **数据集总大小**: 4.27 GB (4,374.53 MB)
- **文件格式**: NumPy 压缩数组 (.npz)
- **文件命名**: `BDMAP_########.npz` (例如: BDMAP_00000001.npz),编号可能不连续


## 数据结构

每个 `.npz` 文件包含 6 个 NumPy 数组：

### 1. `sensor_pc` - 输入点云
- **描述**: 部分人体表面点云数据（传感器采集）
- **形状**: `(N, 3)` - N 为点云数量，每个点包含 (x, y, z) 坐标
- **数据类型**: `float32`
- **坐标系**: 世界坐标系（单位：毫米）
- **点云数量范围**: 约 24,000 ~ 105,000 个点（因样本而异）
- **示例形状**:
  - `(104522, 3)` - 104,522 个点
  - `(24798, 3)` - 24,798 个点
  - `(55356, 3)` - 55,356 个点

### 2. `voxel_labels` - 体素类别标签（目标输出）
- **描述**: 完整人体的三维体素网格，每个体素标注了解剖结构类别
- **形状**: `(X, Y, Z)` - 三维网格，尺寸因样本而异
- **数据类型**: `uint8`
- **类别数量**: 70 个解剖结构类别（ID: 0-69）
- **体素分辨率**: 4mm × 4mm × 4mm
- **网格尺寸范围**:
  - X 轴: 78 ~ 117 体素
  - Y 轴: 56 ~ 92 体素
  - Z 轴: 54 ~ 241 体素


### 3. `grid_world_min` - 网格最小世界坐标
- **描述**: 体素网格在世界坐标系中的最小边界点
- **形状**: `(3,)` - [x_min, y_min, z_min]
- **数据类型**: `float32`
- **单位**: 毫米 (mm)

### 4. `grid_world_max` - 网格最大世界坐标
- **描述**: 体素网格在世界坐标系中的最大边界点
- **形状**: `(3,)` - [x_max, y_max, z_max]
- **数据类型**: `float32`
- **单位**: 毫米 (mm)

### 5. `grid_voxel_size` - 体素尺寸
- **描述**: 每个体素的物理尺寸
- **形状**: `(3,)` - [voxel_x, voxel_y, voxel_z]
- **数据类型**: `float32`
- **固定值**: `[4.0, 4.0, 4.0]` mm - 所有样本统一使用 4mm 分辨率

### 6. `grid_occ_size` - 网格占用尺寸
- **描述**: 体素网格的维度大小
- **形状**: `(3,)` - [size_x, size_y, size_z]
- **数据类型**: `int32`
- **说明**: 与 `voxel_labels` 的形状一致

## 数据格式版本

- **格式版本**: 3.0
- **坐标系**: 世界坐标系（单位：毫米）
- **标签类型**: `uint8` (0-255)
- **点云类型**: `float32`

## 注意事项

1. **文件命名**: 所有文件遵循 `BDMAP_########.npz` 格式，编号可能不连续
2. **动态网格**: 每个样本的 `voxel_labels` 形状不同，需要动态处理或填充
3. **坐标对齐**: `sensor_pc` 和 `voxel_labels` 使用相同的世界坐标系
4. **类别映射**: 类别 ID 与类别名称的对应关系见 `Dataset/dataset_info.json`
5. **数据完整性**: 所有 10,779 个文件均包含完整的 6 个数组
