1. 项目定位与一句话原理

VoxTell（CVPR 2026）是一款 3D 视觉-语言分割模型，它把任意自由文本（从单词「liver」到整句「spiculated carcinoma in the left upper lobe with pleural contact」）直接映射为体素级二值掩膜。在 11 个未见过的 CT/MRI/PET 数据集上取得 SOTA，并在 ReXGroundingCT 报告级基准上拿到第一名。

一句话原理： 图像走 nnU-Net 风格的 3D ResEncL 主干 → bottleneck 作为 memory，文本走冻结的 Qwen3-Embedding-4B → 作为 query，6 层 DETR-style Transformer 把文本「问」进图像，得到每条 prompt 一个 mask embedding；该向量被投影到 U-Net 解码器的 5 个尺度上，通过 einsum 与特征图做点积（MaskFormer 多尺度版），直接生成对应该 prompt 的分割 logits。本质上 prompt 就是一组动态 1×1×1 卷积核。

核心贡献 ①

大规模 3D 多模态数据：158 数据集 / 62K+ 体素图 / 1087 概念 / 9682 改写标签。

核心贡献 ②

多阶段视觉-语言融合：在解码器 5 个尺度全部注入文本，而非只在最后一层。

核心贡献 ③

SOTA 文本可提示 3D 分割：未见类别、跨模态、临床句子级 prompt 均显著领先。

3. 模型架构详解

3.1 图像编码器：ResidualEncoder（6 stage）

来自 dynamic_network_architectures，nnU-Net 同款 ResEncL。参数源自 plans.json：

patch_size           = [192, 192, 192]
features_per_stage   = [32, 64, 128, 256, 320, 320]
strides              = [1, 2, 2, 2, 2, 2]   # 每阶 ↓2 (除 stem)
n_blocks_per_stage   = [1, 3, 4, 6, 6, 6]
kernel_sizes         = 6×[3,3,3]
norm                 = InstanceNorm3d
nonlin               = LeakyReLU(inplace=True)

编码器 return_skips=True，因此输出 6 个尺度的 skip 特征图：
192³×32 → 96³×64 → 48³×128 → 24³×256 → 12³×320 → 6³×320。

3.2 文本编码器：冻结的 Qwen3-Embedding-4B

由 VoxTellPredictor.__init__ 默认加载：

text_encoding_model = 'Qwen/Qwen3-Embedding-4B'
self.tokenizer       = AutoTokenizer.from_pretrained(text_encoding_model, padding_side='left')
self.text_backbone   = AutoModel.from_pretrained(text_encoding_model).eval()

每条 prompt 走 wrap_with_instruction() 包裹：

Instruct: Given an anatomical term query, retrieve the precise
anatomical entity and location it represents
Query: {your_prompt}

经过 Qwen3 后取 last-token pooling（left-padding 时取 [:, -1]，否则按 attention_mask 取最后一个非-padding 的 hidden state），输出 2560 维向量。

训练 / 推理时 Qwen3 全程冻结、且只在算 embedding 时上 GPU。在 predictor.py:189-205，调用完立刻 self.text_backbone.to('cpu') 让出显存。论文还指出，所有训练用 embedding 是提前预计算的，避免每次 forward 都过一次 LM。

3.3 Prompt Decoder：6 层 DETR-style Transformer

定义在 voxtell_model.py:191-202：

角色张量形状（推理时）来源
query (tgt)文本 embedding(N, B, 2048)Qwen3 2560 → MLP → 2048
memory图像 bottleneck(12·12·12, B, 2048)encoder stage4 320 → MLP → 2048
pos embed3D 正弦位置编码(12³, 1, 2048)注册为 buffer（voxtell_model.py:187）

结构超参（参见 VoxTellModel.TRANSFORMER_* 常量）：

  • num_layers = 6，nhead = 8，d_model = 2048
  • 使用 pre-norm（normalize_before=True）：先 LayerNorm 再 attention/FFN，训练更稳。
  • 每层：self-attn(tgt) → cross-attn(tgt as Q, memory as K,V) → FFN（dim_feedforward=2048, dropout=0.1, ReLU）。

6 层 cross-attn 之后，每条 prompt 收敛为一个 2048-d 「mask embedding」。

3.4 多尺度投影头：把一个向量变成 5 个动态卷积核

同一个 2048-d mask embedding 被 5 个独立 MLP（project_to_decoder_channels，voxtell_model.py:171-183）投到解码器各尺度通道数，num_heads=32：

stage 0 (192³) ← 32 ch (final seg head) stage 1 ( 96³) ← 64 · num_heads = 32 (extra channels) stage 2 ( 48³) ← 128 · 32 stage 3 ( 24³) ← 256 · 32 stage 4 ( 12³) ← 320 · 32

3.5 Image Decoder：MaskFormer 多尺度版

核心在 VoxTellDecoder.forward()（voxtell_model.py:405-472）。从 bottleneck 向上一层层 upsample + skip concat，关键的两类操作：

(a) 中间层：作为 32 个额外通道注入

fusion = einsum('b c h w d, b n c -> b n h w d',
                feat, mask_embed.view(B, num_heads=32, C//32))
feat   = torch.cat([feat, fusion], dim=1)   # 32 head = 32 个新通道

(b) 最末层：直接生成分割 logits

seg = einsum('b c h w d, b n c -> b n h w d', feat, mask_embed)

论文 Eq.(5) 形式化为：

y_s = concat( z'_s , T_s · z'_s ), z'_s ∈ R^(C_s × H_s × W_s × D_s) T_s ∈ R^(G × C_s) G = 32

3.6 Deep Supervision

训练时，5 个解码尺度上的预测都参与监督，目标 mask 被下采样到对应分辨率：

L = Σ_{s=1..S} λ_s · L_seg( Ŷ_s , Y_s ),       λ_s = [1, 1/2, 1/4, 1/8, 1/16]
L_seg = Dice + BCE
deep supervision 的作用：强迫早期解码层就开始消化文本条件，否则单层晚融合无法学到 prompt-aware 表征。论文 Tab.2 显示 5-stage fusion + deep sup 比单层 MaskFormer 高 7+ Dice。

4. 推理细节

4.1 预处理（predictor.py:110）

  1. RAS 重定向：通过 nnU-Net 的 NibabelIOWithReorient 读入，确保左右肾、左右肺正确。
  2. 非零裁剪：crop_to_nonzero 去掉外围空气区域。
  3. Z-score 归一化：ZScoreNormalization。不做 spacing resample。

4.2 滑窗推理（predictor.py:208）

  • patch 192³，tile_step_size = 0.5 即 50% 重叠。
  • 每个 patch 输出乘以一个中心权重高的 3D Gaussian（sigma_scale=1/8, scale=10），最后除以累积权重，抑制拼接边界伪影。
  • 用 Thread + Queue(maxsize=2) 生产者-消费者并行：CPU 切 patch / GPU 推理重叠。
  • autocast 半精度，logit 累加用 torch.half 省显存。
  • 原图小于 patch_size 时自动 pad_nd_image 补零。

4.3 后处理

sigmoid > 0.5 → 每条 prompt 独立二值掩膜 → insert_crop_into_image 还原到原始坐标，输出 shape (N, X, Y, Z)。

4.4 三种调用方式

CLI（推荐）

voxtell-predict \
    -i case001.nii.gz \
    -o output_folder \
    -m /path/to/voxtell_v1.1 \
    -p "liver" "right kidney" "spleen"     # 1..N 条 prompt
# 默认每条 prompt 一个文件，--save-combined 合并为多标签

Python API

from voxtell.inference.predictor import VoxTellPredictor
from nnunetv2.imageio.nibabel_reader_writer import NibabelIOWithReorient

img, _ = NibabelIOWithReorient().read_images(["case.nii.gz"])
pred   = VoxTellPredictor(model_dir="voxtell_v1.1", device=device)\
            .predict_single_image(img, ["liver", "right kidney"])
# pred.shape == (2, X, Y, Z)

napari 插件

仓库提供 napari-voxtell 插件，可交互调用同一推理器。

⚠️ 关键陷阱
  • 方向：必须 RAS。若 prompt "liver" 误分割到脾，多半是方向问题。
  • 体素间距：模型不做 spacing 归一化。脑 MRI 高分辨率（如 0.5mm）可能效果差，建议先重采样到 1.5mm 等典型值。
  • PyTorch 版本：2.9.0 有 3D conv OOM bug，请固定 torch==2.8.0。

5.2 每个 batch 怎么采 prompt（关键 trick）

论文 Appendix A.3：

  • 每张训练 patch 同时查询 3 条文本 prompt（在 A100/40GB 上为 batch=2 时的极限）：
    • 2 条 positive：当前 patch 中确实存在的结构。
    • 1 条 negative：图中不存在的结构（如脑 MRI 配 "lung"）。负样本用于教会模型「该 prompt 没有对象就输出空 mask」。
  • Foreground oversampling 85%：85% 概率强制采到含目标的 patch，否则随机。
  • Synonym sampling：每个 positive prompt 都从 9682 改写表里随机抽，25% 用默认主名（如 "liver"）、75% 用改写变体（如 "hepatic organ"）。
为什么必须有 negative prompt？
否则模型会学到「无论给什么 prompt 都给一个像样的前景」。VoxTell 通过 sigmoid 输出独立的二值 mask，必须明确区分「真没有」和「我不会」。

5.3 文本嵌入预计算

论文："During training, all text embeddings are pre-computed for efficiency."
也就是说，9682 条改写标签 + 各 instance prompt 全部离线过一遍 Qwen3 缓存成向量。Dataloader 里只查 dict，从不调用 LM。仓库当前推理代码 (embed_text_prompts) 把这一步搬到 GPU 在线算；训练版本需要写一个预计算脚本。

6.2 数据格式与目录约定

遵循 nnU-Net v2 的「raw → preprocessed → trained」三层目录。文件名约定见 NibabelIOWithReorient：

nnUNet_raw/
└── Dataset800_VoxTell/
    ├── imagesTr/
    │   ├── case_0001_0000.nii.gz   # _0000 是通道 0（CT 单通道）
    │   ├── case_0002_0000.nii.gz
    │   └── ...
    ├── labelsTr/
    │   ├── case_0001.nii.gz        # 多标签（每个数据集自己的整数 ID）
    │   ├── case_0002.nii.gz
    │   └── ...
    └── dataset.json                # nnU-Net 元数据 + 每个 label-id 的语义名
VoxTell 与标准 nnU-Net 的差别：
① 网络头不再是 (B, num_classes, ...) 而是 (B, num_prompts, ...) 的独立 sigmoid，因此训练 label 在采样时被即时拆成「one-vs-rest binary mask」。
② dataset.json 里每个 label-id 必须能映射到一组文本同义词列表（即词汇表里的「主名 + 改写变体」），dataloader 在采 prompt 时按此查表。

6.4 词汇表构造（论文 Appendix C）

这是 VoxTell 能「听懂同义词、拼写错误、句子级 prompt」的根本原因，比模型结构更值得借鉴：

  1. 语义聚合：把每个数据集的实例标注合并成 base label（如所有 tumor instance → 一个 "tumor"）。
  2. 语义扩展：把 base label + 数据集论文 + challenge 文档 + SNOMED-CT 编码喂给 Claude Sonnet 4（论文用的就是 Anthropic Claude），生成：
    • 单标签同义词（如 "right kidney" → "right renal organ"、"dexter kidney"）
    • 组合标签（"left rib 1"…"left rib 12" → "left rib cage"）
    • 语言/医学拼写变体、缩写、错别字
  3. 跨数据集统一：解决同义异名 / 同名异义（典型例子：「liver」在不同数据集是否包含病灶？需要明确定义）。
  4. 人工校验：放射科专家最终签核。

最终落地为一张 label_name → 文本变体列表 的字典，训练时按 25/75 比例采主名/变体。

6.5 Instance-Focused 数据集（用于 ReXGroundingCT）

覆盖「报告里写的某个病灶」这种实例级查询，由三部分组成：

  • (i) ReXGroundingCT 官方训练集（报告→3D mask）；
  • (ii) 把语义分割数据集用 TotalSegmentator 自动转成实例标注；
  • (iii) 含结构化 DICOM 元数据的 TCIA 集合（如 RADCURE、BrainGammaKnife）—— metadata 转成空间描述 prompt。

7. 如何从零启动训练（实操指引）

注意：当前发布的 voxtell/ 包只含推理代码。论文 README 的 Roadmap 也明确写道：「Fine-Tuning：Support and scripts for custom fine-tuning」尚未发布（[ ]）。下面是基于论文 + nnU-Net 复用的可行路线。

7.1 环境

conda create -n voxtell python=3.12 && conda activate voxtell
pip   install torch==2.8.0 torchvision==0.23.0 \
              --index-url https://download.pytorch.org/whl/cu126
pip   install -e .                   # voxtell + nnunetv2 等依赖

7.2 数据准备 Checklist

  1. 把所有数据集统一转换为 nnU-Net v2 的 nnUNet_raw/DatasetXXX_*/ 结构（imagesTr/_0000.nii.gz + labelsTr/.nii.gz + dataset.json）。
  2. 所有图像 reorient to RAS。VoxTell 强依赖此朝向。
  3. 构建 label-id → 文本变体 词汇表（可以用 Claude / GPT 自动生成 + 人工抽查），通常存为一个独立 JSON。
  4. 用脚本把所有文本变体过一次 Qwen3-Embedding-4B，缓存为 text_embed_cache.pt（key = 文本字符串，value = 2560-d tensor）。

7.3 计算量预算（粗略）

  • 每条 prompt 一次完整 forward：约 1.5–2 GB 显存（patch 192³, fp16）。
  • 3 条 prompt × batch 2 → 单 A100 40GB 刚好。
  • 2000 epoch × 250 iter ≈ 50w iter。单卡 A100 实测约 3–4 周。
  • 切到多卡：DDP，batch=128 需要 64 张 A100（论文配置），或者用 grad accumulation 在 8 张上模拟 batch=64。

7.4 训练循环关键代码骨架

for patch, label_volume, present_label_ids in dataloader:
    # 1) 采 prompt
    pos_ids = random.sample(present_label_ids, k=2)
    neg_id  = random.choice([l for l in all_label_ids if l not in present_label_ids])
    prompts = [sample_variant(i, p_main=0.25) for i in pos_ids + [neg_id]]   # 3 条

    # 2) 查缓存
    text_embed = torch.stack([text_embed_cache[p] for p in prompts])  # (3, 2560)

    # 3) 构造 one-vs-rest binary mask
    gt_masks = torch.stack([(label_volume == i).float() for i in pos_ids]
                          + [torch.zeros_like(label_volume)])     # (3, D, H, W)

    # 4) forward + deep supervision loss
    preds = model(patch, text_embed.view(1, 3, 1, -1))   # list of 5 scales
    loss  = sum(λ_s * (dice(p_s, ds(gt_masks, s)) + bce(p_s, ds(gt_masks, s)))
                for s, p_s in enumerate(preds))

7.5 验证 / 评测

  • 每张验证图遍历其全部 GT label-id，对每个 id 取主名作为 prompt，计算 Dice、HD95、IoU。
  • ReXGroundingCT 场景下额外报 HIT5%（Dice ≥ 5% 的样本比例）。

8. 可改造的关键插口

① 替换文本编码器

改 VoxTellPredictor.__init__ 的 text_encoding_model 与构造时的 text_embedding_dim（当前 2560，对应 Qwen3-4B）。投影头 project_text_embed 自动适配新维度。
论文 Appendix D 比较了多种 MTEB 模型，发现 mid-size 编码器在性价比上最优。

② 调整融合阶段数

num_maskformer_stages（1..5）。设为 1 退化为 MaskFormer 单层晚融合。论文 Tab.2 显示 1→3→5 阶段单调涨点。

③ 改 prompt 模板

wrap_with_instruction() 的 Instruct 串显著影响 Qwen3 embedding 语义聚焦，可拆分为「解剖 / 病变 / 位置」多模板。

④ 选哪一层做 memory

decoder_layer（默认 4，即 12³×320 bottleneck）。浅层更高分辨率改善小目标但显存暴涨；DECODER_CONFIGS 已枚举 0–5。

⑤ 注入方式

核心在 VoxTellDecoder.forward 的两次 einsum。可替换为 FiLM、cross-attn、可学习偏置等。

⑥ 负样本比例

当前 2 pos + 1 neg。增加 neg 比例可能改善「prompt 不存在时输出空 mask」的稳定性，但太高会损失正样本梯度。
